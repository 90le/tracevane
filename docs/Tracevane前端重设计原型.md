# Tracevane 前端重设计原型：统一 Agent 工作台

> 状态：Active prototype baseline  
> 更新：2026-06-25  
> 目的：推翻旧版“三栏管理台 / 卡片堆 / OpenClaw 专属对话”的前端方向，重新约定 Tracevane 的页面原型、信息架构和交互边界。本文先于代码实现，后续 UI 修改必须以本文为主。

## 0. 研究记录

本轮原型不是凭空设计，而是对齐当前主流 Agent 产品形态：

- OpenAI Codex CLI 官方文档说明 Codex 是可在本机目录读取、修改并运行代码的本地 coding agent，具备交互、模型切换、脚本化、MCP、approval modes、subagents 等能力；Tracevane 不能把 Codex 简化成“一个聊天框”。来源：`https://developers.openai.com/codex/cli`，2026-06-25 checked。
- Anthropic Claude Code CLI 官方文档显示 Claude Code 支持 interactive session、print mode、continue/resume、agent、tools allow/deny、permission mode、MCP、plugins、background session 等运行控制；Tracevane 的 Agent 会话必须能承载权限、恢复、工具事件和运行证据，而不是固定 OpenClaw 对话。来源：`https://code.claude.com/docs/en/cli-usage`，2026-06-25 checked。
- OpenCode 官方站点强调 multi-session、plan/edit/run/validate/debug、desktop tabs、diff viewer、background agents、model agnostic；Tracevane 的前端不应该按单一供应商分裂会话，而应该做统一 Agent Session。来源：`https://www.opencode.asia/`，2026-06-25 checked。

## 1. 推翻旧设计

### 1.1 废弃的旧模型

旧模型可以概括为：

```text
左导航 + 主列表 + 右详情 / Inspector
```

这个模型在少量管理页可用，但不适合作为 Tracevane 的整体设计理念。它导致：

- 每个域都长成类似“三栏后台”；
- 对话、IM 投递、OpenClaw 原生会话、CLI runs 重复出现；
- 右侧 Inspector 被滥用，变成“什么都塞一点”；
- 页面像说明文档或状态卡片，而不是可以工作的控制台；
- 移动端天然崩坏，只能横向挤压或隐藏关键内容。

### 1.2 新模型

新版前端采用：

```text
Global Shell
  └─ Domain Console
       ├─ Command Bar / Context Strip
       ├─ Primary Stage
       ├─ Local Switcher
       ├─ Inline Evidence Rail（按需展开）
       └─ Modal / Drawer / Bottom Sheet（临时任务）
```

核心区别：

- **只有一个主舞台 Primary Stage**，每个页面打开后 3 秒内能知道主对象是什么。
- **详情不常驻右侧**，只有用户选中对象、查看证据、编辑配置时才通过 Drawer / Sheet / 子页面出现。
- **局部导航横向或分段显示**，不再左侧套左侧、右侧套右侧。
- **列表、时间线、表格、配置页各自选合适形态**，不是所有页面都强行三栏。
- **移动端是同一信息架构的单栏任务流**，不是被压扁的桌面版。

## 2. 新产品信息架构

一级导航保持简短：

```text
总览
工作
  工作区
  Agent 会话
  CLI 代理
连接
  模型网关
  IM 渠道
平台
  平台
```

### 2.1 页面职责

| 页面 | 主对象 | 页面形态 | 明确不做 |
| --- | --- | --- | --- |
| 总览 | 当前系统能不能工作、下一步去哪 | 状态指挥台 + 最近任务流 | 不平铺每个域的 CRUD |
| 工作区 | 本地项目工作面 | 文件/编辑/终端/预览的可切换工作台 | 不管理 CLI runtime 安装 |
| Agent 会话 | 所有 Agent 的统一会话 | 会话时间线 + 上下文控制条 + 证据抽屉 | 不做 OpenClaw 专属聊天 |
| CLI 代理 | 本地 Agent runtime 与 runs | runtime 表 + run 队列 + 详情抽屉 | 不做普通终端，不做聊天 |
| 模型网关 | Provider/model/protocol/client route | 路由矩阵 + Provider/模型子页 | 不显示 Agent 会话内容 |
| IM 渠道 | IM 账号、Bot、绑定、投递 | 平台账号 / 绑定路由 / 投递 / 守护 | 不承担会话 UI |
| 平台 | 第三方平台原生管理 | 平台目录 + 子平台工作台 | 不拥有模型网关/IM/CLI 的写入口 |

## 3. 全局壳层原型

### 3.1 桌面

```text
┌──────────────────────────────────────────────────────────────────────────────┐
│ Tracevane  [⌘K 搜索/跳转/命令]                         主题  状态  用户/本机 │
├──────────────┬───────────────────────────────────────────────────────────────┤
│ 总览         │ 面包屑 / 页面标题 / 当前上下文                         动作区 │
│ 工作         ├───────────────────────────────────────────────────────────────┤
│  工作区      │                                                               │
│  Agent 会话  │                  Domain Primary Stage                         │
│  CLI 代理    │                                                               │
│ 连接         │                                                               │
│  模型网关    │                                                               │
│  IM 渠道     │                                                               │
│ 平台         │                                                               │
│  平台        │                                                               │
└──────────────┴───────────────────────────────────────────────────────────────┘
```

规则：

- 全局侧栏只负责跨域跳转，不再承载二级/三级业务对象。
- 页面内部使用顶部局部 switcher / tabs / 子路由，不再在页面内再造一套永久左侧栏。
- Command Palette 是跨域动作入口：新建会话、切换模型、打开工作区、添加 IM 账号、检查 CLI 等。

### 3.2 移动端

```text
┌──────────────────────────┐
│ Tracevane     ⌘K   状态  │
├──────────────────────────┤
│ 页面标题 / 当前上下文     │
├──────────────────────────┤
│ Local Switcher           │
├──────────────────────────┤
│ Primary Stage            │
│                          │
├──────────────────────────┤
│ 总览 工作 连接 平台       │
└──────────────────────────┘
```

规则：

- 移动端只有一个主内容流。
- 所有 Drawer 变 Bottom Sheet 或全屏编辑页。
- 表格必须自动变为 row list，不允许页面级横向滚动。

## 4. Agent 会话原型

### 4.1 定位

Agent 会话是 Tracevane 的统一人机交互层：

```text
Web 输入 / IM 消息 / CLI Run / OpenClaw 原生 Agent / 未来平台
      ↓
Tracevane Agent Session
      ↓
Codex / Claude Code / OpenCode / OpenClaw / custom agent
```

OpenClaw 不能拥有单独的对话中心；OpenClaw 只是 Agent 来源之一。

### 4.2 桌面原型

```text
┌──────────────────────────────────────────────────────────────────────────────┐
│ Agent 会话                                                                  │
│ [新建会话] [来源: 全部] [Agent: 全部] [状态: 运行/等待/失败]       搜索会话   │
├──────────────────────────────────────────────────────────────────────────────┤
│ Context Strip                                                               │
│ Agent: Codex ▾   模型: gpt-5.5 ▾   目录: ~/project ▾   权限: Plan/Ask ▾     │
├──────────────────────────────────────────────────────────────────────────────┤
│ 会话列表 / 时间排序                                                         │
│ ┌──────────────────────────────────────────────────────────────────────────┐ │
│ │ 法律咨询合同风险      Codex · gpt-5.5 · 飞书私聊 · 等待回复 · 2 分钟前   │ │
│ │ 修复网关 502          Claude Code · 网页 · 正在运行 · 12 分钟前          │ │
│ │ OpenClaw 绑定检查     OpenClaw Agent · 平台触发 · 已完成                │ │
│ └──────────────────────────────────────────────────────────────────────────┘ │
├──────────────────────────────────────────────────────────────────────────────┤
│ 当前会话：法律咨询合同风险                                                  │
│ 来源：飞书私聊 / 张三    工作目录：~/.openclaw/法律咨询    会话 ID [复制]    │
│                                                                              │
│ 你：帮我分析合同风险                                                         │
│ Codex：我会读取合同并总结主要风险。                                          │
│ 工具：读取文件 合同.docx        [查看详情]                                   │
│ Codex：发现 3 个风险：付款逾期、解除条款、争议解决地。                      │
│                                                                              │
│ ┌──────────────────────────────────────────────────────────────────────────┐ │
│ │ 输入消息，@文件，/命令，拖入附件                                       │ │
│ └──────────────────────────────────────────────────────────────────────────┘ │
│ [发送] [附加上下文] [转入工作区] [停止]                         [证据抽屉] │
└──────────────────────────────────────────────────────────────────────────────┘
```

关键点：

- 默认不是左右三栏，而是 **会话索引 + 当前会话舞台** 的上下分层。
- 会话列表显示人类标题，不显示长 session key。原始 key 只在详情中复制。
- Context Strip 是会话的关键控制面：Agent / 模型 / 目录 / 权限必须可见可切换。
- 工具调用默认人话化，原始 JSON 进详情抽屉。
- 证据、diff、日志、IM 投递记录通过 Evidence Drawer 临时打开，不常驻抢空间。

### 4.3 移动端原型

```text

│ Codex · gpt-5.5 · Plan   │

│ 当前会话消息流            │

├──────────────────────────┤

```

## 5. 模型网关原型

模型网关不做三栏 Provider admin，而做“模型服务控制台”。

```text
┌──────────────────────────────────────────────────────────────────────────────┐
│ 模型网关                                                                    │
│ [服务状态] [Provider] [模型] [客户端接入] [用量] [诊断]                      │
├──────────────────────────────────────────────────────────────────────────────┤
│ 当前路由健康：正常  Provider 3  模型 28  协议 openai/responses/anthropic    │
├──────────────────────────────────────────────────────────────────────────────┤
│ 路由矩阵                                                                     │
│ Client       Alias       Provider/Model        Protocol        Status        │
│ Claude Code  gpt-5.5     Codex Account/gpt-5.5 anthropic       OK            │
│ Codex CLI    gpt-5.5     Codex Account/gpt-5.5 responses       OK            │
│ OpenCode     glm-5.2     GLM/glm-5.2         openai           OK            │
│                                                                              │
│ 行操作：检查 / 复制接入 / 查看 fallback / 打开用量                           │
└──────────────────────────────────────────────────────────────────────────────┘
```

详情通过 Drawer：Provider 编辑、模型上下文、fallback 证据、协议转换证据。

## 6. IM 渠道原型

IM 渠道不是“聊天页”，只负责外部消息入口和投递证据。

```text
┌──────────────────────────────────────────────────────────────────────────────┐
│ IM 渠道                                                                     │
│ [平台账号] [绑定路由] [会话投递] [队列策略] [守护诊断]                       │
├──────────────────────────────────────────────────────────────────────────────┤
│ 绑定路由                                                                     │
│ 来源             Agent        模型        目录              权限    状态     │
│ 飞书私聊: 张三   Codex        gpt-5.5     ~/法律咨询        Plan    启用     │
│ 飞书群: 研发群   Claude Code  默认        ~/workspace       Ask     启用     │
│                                                                              │
│ [新建绑定] [复制] [删除] [检查]                                              │
└──────────────────────────────────────────────────────────────────────────────┘
```

投递页只显示：

- 哪个 IM 会话触发；
- 被路由到哪个 Agent Session；
- 是否排队、失败、已回复；
- 点击后进入 Agent 会话详情。

## 7. CLI 代理原型

CLI 代理是 runtime 控制台，不是终端、不是聊天。

```text
┌──────────────────────────────────────────────────────────────────────────────┐
│ CLI 代理                                                                    │
│ [运行时] [Agent Runs] [配置检查] [安装修复]                                  │
├──────────────────────────────────────────────────────────────────────────────┤
│ Runtime Readiness                                                           │
│ Agent        Version       Auth        Gateway        Actions                │
│ Codex        0.x           已登录      OK             检查 / 修复 / 打开会话 │
│ Claude Code   2.x          已登录      OK             检查 / 修复            │
│ OpenCode      1.x          未安装      —              安装                   │
├──────────────────────────────────────────────────────────────────────────────┤
│ Agent Runs                                                                 │
│ 时间        来源       Agent        模型        状态       结果              │
│ 12:30       IM         Codex        gpt-5.5     完成       查看证据          │
│ 12:18       Workspace  Claude       默认        失败       查看错误          │
└──────────────────────────────────────────────────────────────────────────────┘
```

## 8. 平台 / OpenClaw 原型

平台是第三方平台目录，OpenClaw 是一个平台子应用。

```text
平台
  OpenClaw
    总览
    守护
    配置
    原生 Agent
    Channel / Bot
    原生绑定
    Skills
    服务
    日志
    诊断
```

OpenClaw 内部仍然不能做“OpenClaw 独立对话”。如果用户从 OpenClaw Agent 发起对话，进入统一 Agent 会话，并带上来源标签：`platform:openclaw`。

## 9. 交互模式

| 场景 | 使用方式 | 禁止 |
| --- | --- | --- |
| 对象列表 | 单列表 / 表格 / 分组列表 | 固定左栏 + 固定右栏详情 |
| 创建/编辑配置 | Drawer 或子页面 | 在列表下方展开超长表单 |
| 查看日志/JSON/证据 | Evidence Drawer / Sheet | 常驻右侧占屏 |
| 危险操作 | Dialog 二次确认 | Toast 里直接完成 |
| 大型配置 | 子页面 + 保存条 | 一屏堆 30 个输入框 |
| 移动端详情 | Bottom Sheet / 全屏页 | 横向滚动三栏 |

## 10. 实施优先级

P0：文档与导航语言统一。

- `/chat` 前端命名改为 Agent 会话。
- 文档删除“OpenClaw 对话中心”概念。
- IM 投递点击进入 Agent 会话，而不是在 IM 内完整复刻会话。
- OpenClaw 平台不出现独立聊天入口。

P1：Agent 会话重构。

- 会话列表人话化标题。
- 会话管理使用右键菜单 / 更多菜单，不在列表行常驻重命名、归档、删除等按钮。
- 会话筛选覆盖活跃、自管、只读、归档、全部和 organizer 文件夹；完整文件夹/子文件夹管理进入独立 Organizer 面板或 Drawer。
- Context Strip 支持 Agent / 模型 / 目录 / 权限。
- 统一来源：web / im / cli / platform。
- Evidence Drawer 承载 raw event、diff、tool、IM 投递。

P2：页面收敛。

- 模型网关、IM、CLI、平台统一使用 Domain Console，不再各做三栏后台。
- 移动端单栏验收。
- 表格移动端 row list 化。

## 11. 验收标准

- 打开任何页面，3 秒内能说清主对象。
- 页面没有固定三栏强行塞满的布局。
- OpenClaw 没有独立对话页；对话统一在 Agent 会话。
- IM 会话投递只做路由/证据，不做完整聊天 UI。
- CLI 代理没有普通 PTY 控件。
- 模型网关不展示 Agent 消息时间线。
- PC 端无页面级横向滚动；移动端无横向溢出。
