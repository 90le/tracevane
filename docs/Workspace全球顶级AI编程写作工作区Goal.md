本文件保留为旧链接兼容入口。此前把写作列为当前主线的表述已经被纠正：

- **当前主线不是独立写作产品线。**
- 当前主线是 **IDE-first / AI 编程 IDE 工作区 / 可扩展 Web IDE 平台**，优先完成工作区本体、终端、前后端边界、桌面/手机 UI/UX。
- 写作、渲染、预览增强只保留为未来可选扩展能力，不进入当前 Phase 0/1/2 的目标、验收或重构范围。

后续所有目标、清理、架构、路线评估、桌面/平板/手机响应式要求、提交约束与验收标准，均以：

- `Workspace全球顶级AI编程IDE工作区Goal蓝图.md`

为准。

## 当前必须执行的方向

1. 先清理不恰当、过期、误导或冲突的前后端 IDE 工作区代码与文档。
2. 恢复并巩固真实 IDE 能力：文件、编辑器、终端、Git、搜索、命令、布局恢复。
3. AI 作为编程协作层进入 IDE：上下文、diff、命令审批、证据闭环。
4. 桌面、平板、手机分别设计，不把手机端做成缩小版桌面。
5. 在 Theia、code-server/OpenVSCode、当前自研 Workbench 三条路线之间完成 Spike 和 ADR 后，再进入系统重建。
6. 不再推进说明页式、卡片墙式、概念海报式 Workspace。

## 明确不要做

- 不建设独立写作产品线。
- 不做渲染/预览增强。
- 不把写作列为当前主线任务。
- 不把旧的“编程 + 写作并列主线”文案作为当前产品验收。
- 不用旧原型文档覆盖新的 IDE-first Goal 蓝图。
