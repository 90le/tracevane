# 实施阶段、验收与风险

## 1. 总体实施策略

推荐按照“先可靠编辑，再独立工作台，再终端，再语言服务”的顺序。

不要从第一天就追求完整 VS Code。

正确节奏：

```txt
M1：文件管理器在线编辑器基础版
M1.x：文件管理器在线编辑器增强版
M2：独立 IDE 基础工作台
M3：终端和任务运行
M4：高级编辑与文件同步
M5：LSP、Git、调试等 IDE 能力
```

## 2. M1 文件管理器在线编辑器

### 目标

完成类似宝塔面板在线文本编辑器的能力。

### 范围

```txt
- 文件管理器双击打开文件
- 在线编辑器容器
- Monaco Editor
- 多 Tab
- 保存当前
- 全部保存
- dirty 状态
- 关闭确认
- 搜索替换
- 跳转行
- 字体大小
- 主题切换入口
- 状态栏
```

### 前端交付

```txt
- FileEditorDialog / FileEditorPage
- FileEditorTabs
- MonacoEditorHost
- FileEditorStatusBar
- EditorService
- MonacoModelService
- DirtyStateService
- CommandService 基础版
```

### 后端交付

```txt
- readFile
- writeFile
- stat
- 文件大小限制
- 二进制文件检测
- 权限检测
- version/mtime 返回
```

### 验收标准

```txt
- 能从文件管理器打开文件。
- 能多 Tab 编辑。
- 保存成功后 dirty 清除。
- 保存失败 dirty 保留。
- 关闭未保存文件有确认。
- 同一文件重复打开只激活已有 Tab。
- 切换 Tab 保留光标和滚动位置。
- 大文件和二进制文件有明确提示。
```

## 2.1 M1.x 文件管理器在线编辑器增强

### 目标

在 M1 已验证基础上继续补齐文件管理器编辑器的日常编辑体验和保存安全能力。

### 范围

```txt
- 窗口最大化 / 还原 / 最小化 / 关闭
- Tab 关闭其他 / 关闭全部 / 关闭已保存
- 多 Tab 自适应缩小与横向滚动
- Tab viewState 保留：光标、选区、滚动位置
- 文件 reload / refresh
- 关闭确认：保存 / 不保存 / 取消
- 外部修改冲突检测
- compare / reload / overwrite
- 保存前 mtime/version/hash 校验
- 状态栏增强：换行符、缩进、编码、大小、权限、mtime
- 搜索替换显式选项
- light / dark / auto 主题选择
- 右键菜单、操作列、顶部按钮等编辑入口
```

### 验收标准

```txt
- 打开超过 8 个文件时，不会静默丢弃任何 dirty Tab。
- Tab 顶栏在标签增多时先缩小，达到最小宽度后横向滚动。
- 最大化、最小化、恢复、关闭均有明确且可访问的控件。
- dirty 文件关闭/刷新时可以保存、不保存或取消。
- 外部修改不会被 dirty Tab 静默覆盖。
- 用户可以比较、重新加载或显式覆盖外部修改。
- 状态栏展示的元数据来自真实后端或 Monaco 模型，不猜测。
- 新入口不会让非文本文件误入编辑器。
```

### 与 M2 的边界

M1.x 仍然只增强文件管理器在线编辑器，不引入 Dockview、多编辑组、终端、Git、LSP、Problems 或 Output。

详细执行计划见 `m1x-execution-plan.md`。

## 3. M2 独立 IDE 基础工作台

### 目标

建立独立 IDE 页面和可扩展工作台结构。

### 范围

```txt
- /ide/:workspaceId
- Activity Bar 基础版
- Side Bar Explorer
- Editor Area
- Dockview 接入
- 多编辑组
- Tab 拖动
- Split Right
- Split Down
- Panel Area 框架
- Status Bar
- 布局保存/恢复
- 布局重置
```

### 前端交付

```txt
- IdeWorkbench
- ActivityBar
- SideBar
- ExplorerView
- EditorDock
- EditorGroupView
- PanelArea
- StatusBar
- layoutStore
```

### 后端交付

```txt
- workspace tree
- 文件操作接口复用 M1
- layout 持久化接口可选
```

### 验收标准

```txt
- IDE 是独立入口，不是文件管理器弹窗。
- 左侧资源管理器能打开文件。
- 中间支持多 Tab。
- Tab 能拖拽排序。
- 文件能左右/上下拆分到不同编辑组。
- 布局刷新后可恢复。
- 布局可重置。
```

### 验收标准

## 5. M4 高级编辑与文件同步

### 目标

提升编辑器和 IDE 的工程可用性。

### 范围

```txt

- 全局搜索
- 文件 watcher
- 外部修改提示
- compare / reload / overwrite
- Problems 面板空壳
```

### 验收标准

```txt

```

## 7. 风险清单

| 风险 | 表现 | 影响 | 应对 |
|---|---|---|---|
| 固定布局导致后期重构 | IDE 只能三栏展示，不能拖拽拆分 | 后期接近 VS Code 类体验困难 | 独立 IDE 第一版引入 Dockview |
| 文件管理器和编辑器状态混乱 | 重命名、删除、保存不同步 | 数据丢失、Tab 错乱 | 统一 FileService 和 EditorService |
| 文件内容进 React state | 大文件卡顿、输入延迟 | 编辑体验差 | 内容只放 Monaco model |
| 终端安全风险 | 用户可执行任意命令 | 高危 | 容器隔离、cwd guard、权限控制 |
| 路径逃逸 | 访问 workspace 外文件 | 高危 | 后端 root guard |
| 保存冲突 | 外部修改被覆盖 | 数据丢失 | version/mtime + 409 conflict |
| LSP 复杂度失控 | 多语言服务同时接入 | 进度失控 | 先接单语言 |
| 布局状态不可迁移 | 版本升级后 layout 崩溃 | 用户无法打开 IDE | layout state 加 version 和 reset |
| Monaco model 泄漏 | 打开关闭大量文件后内存升高 | 性能问题 | model 引用计数和 dispose |

## 8. MVP 定义

如果只做最小可用版本，建议 MVP 是：

```txt
MVP-A：文件管理器在线编辑器
- 双击打开文件
- Monaco 编辑
- 多 Tab
- 保存/全部保存
- dirty 确认
- 搜索替换
- 状态栏

MVP-B：独立 IDE 骨架
- /ide/:workspaceId
- 左侧 Explorer
- 中间 Monaco 多 Tab
- Dockview 编辑区
- 底部 Panel 框架
- 布局保存恢复
```

终端不建议放进第一个 MVP，除非后端 PTY 和安全边界已经准备好。

## 10. 上线检查清单

上线前至少检查：

```txt
- 所有文件路径都走后端校验。
- 未保存文件不会静默丢失。
- 保存冲突不会静默覆盖。
- 大文件不会拖垮页面。
- 关闭 Tab 会释放不需要的 model。
- IDE 布局异常可以重置。
- 终端权限受控。
- 删除、重命名、覆盖等危险操作有确认。
- 错误信息能让用户知道下一步怎么处理。
```

## 11. 最终推荐

建议立项时拆成两个 Epic：

```txt
Epic 1：文件管理器在线编辑器
- 目标是快速、可靠、多标签编辑文件。
- 不承担完整 IDE。

Epic 2：独立 IDE 工作台
- 目标是项目级开发工作台。
- 第一版就引入 docking layout。
- 逐步接终端、LSP、Git、任务运行。
```

这样既满足文件管理器的即时编辑需求，也为后续主流 IDE 级体验留出正确架构。
