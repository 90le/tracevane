# 前端实现方案

## 1. 前端目标

前端需要同时支持：

```txt
- 文件管理器在线编辑器
- 独立 IDE 工作台
```

但两者不能互相耦合。正确做法是共享代码编辑内核，不共享页面布局。

## 4. Monaco 接入原则

### Monaco-first 原则

Monaco 已经提供完整编辑器交互，前端不应重复实现单文件编辑器内部能力。Tracevane 的 React 层只做文件生命周期、Tab/window shell、保存安全、冲突处理和状态栏。搜索/替换、上/下一个、大小写、全词、正则、多光标、折叠、minimap、word wrap、context menu、撤销/重做等能力应优先交给 Monaco actions/options/widgets。详细策略见 `10-monaco-first-online-editor-strategy.md`。


### 不要把 Monaco 当 textarea

错误做法：

```txt
- onChange 后每次 setState 保存完整文本。
- Tab 切换时销毁并重建文本。
- 每个组件自己创建 model。
```

正确做法：

```txt
- 一个文件一个 Monaco model。
- 文件内容保存在 model 中。
- React state 只保存 path、dirty、activeTab、viewState。
- 保存时从 model.getValue() 获取文本。
```

### Model URI

建议：

```ts
function createModelUri(workspaceId: string, path: string) {
  return `file:///workspace/${workspaceId}${path.startsWith("/") ? path : `/${path}`}`;
}
```

示例：

```txt
file:///workspace/demo/src/App.tsx
```

### Language 识别

```ts
type LanguageResolver = (path: string, mime?: string) => string;
```

常见映射：

```txt
.js  -> javascript
.jsx -> javascript
.ts  -> typescript
.tsx -> typescript
.json -> json
.css -> css
.html -> html
.md -> markdown
.py -> python
.sh -> shell
.yaml -> yaml
```

## 5. EditorService 伪代码

```ts
async function openFile(path: string, options?: OpenFileOptions) {
  const existing = editorStore.findTabByPath(path);
  if (existing) {
    editorStore.activateTab(existing.id);
    return existing;
  }

  const file = await fileService.readFile({
    workspaceId: currentWorkspaceId,
    path,
  });

  const model = monacoModelService.getOrCreateModel(file);
  const tab = editorStore.addTab({
    path,
    title: getFileName(path),
    language: resolveLanguage(path),
    dirty: false,
    readonly: file.readonly,
  });

  monacoModelService.bindDirtyListener(model, tab.id);
  editorStore.activateTab(tab.id);

  return tab;
}
```

保存：

```ts
async function saveFile(tabId: string) {
  const tab = editorStore.getTab(tabId);
  if (!tab || tab.readonly) return;

  const model = monacoModelService.getModelByPath(tab.path);
  if (!model) return;

  const content = model.getValue();

  await fileService.writeFile({
    workspaceId: tab.workspaceId,
    path: tab.path,
    content,
    expectedVersion: tab.lastSavedVersion,
  });

  editorStore.markClean(tabId);
}
```

## 6. Tab 行为

### 文件管理器编辑器 Tab

M1 基础版已支持打开、激活、关闭和 dirty 标记。M1.x 继续补齐文件管理器编辑器自身的 Tab 体验：

```txt
- 打开
- 激活
- 关闭当前
- 关闭其他
- 关闭全部
- 关闭已保存
- dirty 标记
- dirty-aware 关闭确认
- 超过少量标签后自适应缩小
- 达到最小宽度后横向滚动
- 不因容量限制静默丢弃 dirty Tab
- 切换 Tab 时恢复光标、选区、滚动 viewState
```

文件管理器编辑器仍不支持跨编辑组拖拽、split right/down 或 Dockview 布局；这些属于独立 IDE。

### 独立 IDE Tab

需要增强：

```txt
- 跨编辑组拖动
- 拖动排序
- preview tab
- pinned tab
- split right
- split down
- close to right
- close saved
```

不要让两个产品形态共用完全相同的 Tab UI。

可以共享 Tab 数据模型和关闭确认逻辑。

## 7. 快捷键设计

第一批快捷键：

```txt
Ctrl+S             保存当前文件
Ctrl+Shift+S       保存全部
Ctrl+F             当前文件搜索（Monaco 原生 Find widget）
Ctrl+H             当前文件替换（Monaco 原生 Replace widget）
Ctrl+G             跳转行（可由 Tracevane 薄入口或 Monaco action 触发）
Ctrl+W             关闭当前 Tab
Ctrl+P             快速打开文件，IDE 后续实现
Ctrl+`             切换终端，IDE 后续实现
Ctrl+\             拆分编辑器，IDE 后续实现
```

原则：

```txt
- 快捷键触发 CommandService。
- 不要在多个组件里重复绑定同一个快捷键。
- 输入框、Monaco、终端的快捷键焦点需要区分。
```

## 8. 文件树联动

文件树不直接读写 Monaco。

文件树事件：

```ts
onFileDoubleClick(file) {
  commandService.execute("file.open", { path: file.path });
}

onRename(oldPath, newPath) {
  commandService.execute("file.rename", { oldPath, newPath });
}

onDelete(path) {
  commandService.execute("file.delete", { path });
}
```

EditorService 监听 rename/delete 结果：

```txt
- rename 后更新打开的 Tab。
- delete 后关闭或标记 deleted。
- dirty 文件删除前弹确认。
```

## 9. 布局实现

### 文件管理器在线编辑器

可用普通布局：

```txt
Header
Tabs
Monaco
StatusBar
```

不用 Dockview。

### 独立 IDE

使用 Dockview：

```txt
ActivityBar + SideBar 可以用常规布局。
Editor Area 和 Panel Area 用 Dockview 或 Dockview + SplitView。
```

状态：

```ts
type IdeLayoutState = {
  dockviewLayout: unknown;
  sidebarVisible: boolean;
  panelVisible: boolean;
  activeActivity: ActivityId;
  activePanel: PanelId;
};
```

持久化：

```txt
- localStorage 可用于个人偏好。
- 服务端可用于 workspace layout。
- 布局 state 需要 version。
- 版本不兼容时 reset。
```

## 10. 终端前端

```txt
TerminalView
├─ TerminalTabs
├─ XtermHost
└─ TerminalToolbar
```

关键点：

```txt
- xterm 初始化后 attach WebSocket。
- 面板 resize 后调用 fitAddon.fit()。
- fit 后把 cols/rows 发给后端。
- 关闭 terminal 时通知后端 kill session。
- 组件卸载时 dispose xterm 实例。
```

## 11. 性能要求

需要避免：

```txt
- 大文件内容进 React state。
- 每次输入触发整页 rerender。
- 文件树一次性加载所有目录。
- Monaco model 泄漏。
- 关闭 Tab 后 listener 未释放。
```

建议：

```txt
- 文件树懒加载。
- Monaco model 引用计数。
- 大文件阈值保护。
- Tab 和文件树组件 memo。
- watch event 合并去抖。
```

## 12. 前端验收标准

```txt
- 文件管理器可以打开在线编辑器。
- 独立 IDE 有独立路由。
- 两者共享文件读写和保存逻辑。
- 文件内容不进入全局 React state。
- 多 Tab dirty 状态准确。
- 保存失败不会错误清除 dirty。
- 重命名、删除文件能同步打开中的 Tab。
- IDE 布局可以拖拽、保存、恢复、重置。
- 终端 UI resize 后不出现尺寸错乱。
```
