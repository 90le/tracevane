# 后端服务与接口方案

## 1. 后端目标

前端的 Monaco 和 xterm.js 只负责展示和交互。

真正的能力需要后端提供：

```txt
- workspace 管理
- 文件读写
- 文件树
- 文件搜索
- 文件监听
- 权限校验
- 路径安全
- 终端 PTY
- 任务运行
- 后续 LSP 和 Git
```

## 2. Workspace 边界

所有接口都必须围绕 workspaceId。

```txt
workspaceId
→ 后端解析 rootPath
→ 所有 path 都必须限制在 rootPath 内
```

前端不传绝对路径。

正确：

```json
{
  "workspaceId": "demo",
  "path": "/src/App.tsx"
}
```

不推荐：

```json
{
  "path": "C:\\Users\\xxx\\project\\src\\App.tsx"
}
```

## 3. 路径安全

后端必须做 root guard。

需要防止：

```txt
- ../ 路径逃逸
- URL 编码绕过
- 软链接逃逸
- Windows 盘符路径绕过
- UNC 路径绕过
- 大小写路径绕过
- 删除 workspace root
```

伪代码：

```ts
function resolveWorkspacePath(workspaceRoot: string, userPath: string) {
  const resolved = path.resolve(workspaceRoot, "." + normalizeUserPath(userPath));
  const root = path.resolve(workspaceRoot);

  if (!isSubPath(resolved, root)) {
    throw new Error("Path escapes workspace root");
  }

  return resolved;
}
```

注意 Windows 下要处理大小写和盘符规范化。

## 4. 文件接口

### 获取目录树

```txt
GET /api/workspaces/:workspaceId/tree?path=/src
```

返回：

```ts
type DirectoryNode = {
  path: string;
  name: string;
  type: "file" | "folder";
  size?: number;
  mtime?: number;
  readonly?: boolean;
  hidden?: boolean;
  childrenLoaded?: boolean;
};
```

建议：

```txt
- 目录懒加载。
- 支持排序。
- 支持隐藏文件配置。
- 支持忽略 node_modules、.git 等大目录。
```

### 读取文件

```txt
GET /api/workspaces/:workspaceId/file?path=/src/App.tsx
```

返回：

```ts
type ReadFileResponse = {
  path: string;
  content: string;
  encoding: string;
  eol: "LF" | "CRLF";
  language?: string;
  size: number;
  mtime: number;
  version: string;
  readonly: boolean;
};
```

读取前校验：

```txt
- 是否存在
- 是否为文件
- 是否二进制
- 文件大小是否超过阈值
- 是否有权限
```

### 写入文件

M1.x.2 当前状态：后端读接口已经返回 `modifiedAt`、大小、权限、textLike/truncated 等基础元数据；写接口会保存版本快照，并支持 `expectedModifiedAt` + `expectedSize` 的并发写保护。若用户明确选择强制覆盖，前端传 `force: true` 绕过 expected token 校验。

M1.x.2 同时在前端 active-tab save 之前执行一次当前元数据 preflight，避免在未重新构建后端 dist 或代理到旧服务时只依赖 UI 提示；真实后端契约仍以 409 `file_write_conflict` 为准。

```txt
PUT /api/workspaces/:workspaceId/file
```

请求：

```ts
type WriteFileRequest = {
  path: string;
  content: string;
  encoding?: string;
  eol?: "LF" | "CRLF";
  expectedModifiedAt?: string | null;
  expectedSize?: number | null;
  force?: boolean;
};
```

返回：

```ts
type WriteFileResponse = {
  path: string;
  saved: true;
  modifiedAt: string | null;
  size: number;
};
```

冲突处理：

```txt
如果 expectedModifiedAt / expectedSize 与当前文件版本不一致：
- 返回 409 Conflict。
- 返回当前文件的最新元数据。
- 前端提示文件已被外部修改。
- dirty Tab 不允许静默覆盖。
- 用户可选择重新加载、强制覆盖或取消。
- 前端提供 lightweight local-vs-disk compare；Monaco Diff 仅作为后续视觉增强，不阻塞 M1.x 保存安全闭环。
```

建议响应形态：

```ts
type WriteConflictResponse = {
  code: "file_write_conflict";
  message: string;
  path: string;
  expectedModifiedAt?: string | null;
  expectedSize?: number | null;
  currentModifiedAt: string | null;
  currentSize: number;
};
```

### 新建文件

```txt
POST /api/workspaces/:workspaceId/file
```

请求：

```ts
type CreateFileRequest = {
  path: string;
  content?: string;
  overwrite?: boolean;
};
```

### 新建目录

```txt
POST /api/workspaces/:workspaceId/folder
```

请求：

```ts
type CreateFolderRequest = {
  path: string;
};
```

### 重命名

```txt
PATCH /api/workspaces/:workspaceId/path/rename
```

请求：

```ts
type RenamePathRequest = {
  oldPath: string;
  newPath: string;
  overwrite?: boolean;
};
```

### 删除

```txt
DELETE /api/workspaces/:workspaceId/path?path=/src/App.tsx
```

建议：

```txt
- 默认进入回收站或软删除。
- 危险删除需要二次确认。
- 不允许删除 workspace root。
```


## 4.1 Mini Explorer / IDE Explorer 复用说明

M3 Online Editor Mini Explorer + Shared Explorer Core 不需要新增第二套后端接口。它应复用本章已有文件接口：

```txt
- browse/list directory
- read file / write file
- create file / create directory
- rename
- delete
- copy / move
- stat / metadata
```

前端 `shared/explorer-core` 只做 rootId/path、树状态、排序、过滤和文件操作命令适配；权限、路径逃逸防护、mtime/version/hash 校验仍由后端统一负责。未来独立 IDE Explorer 也复用同一组接口。

### 文件搜索

```txt
GET /api/workspaces/:workspaceId/search?q=keyword&path=/src
```

支持：

```txt
- include glob
- exclude glob
- case sensitive
- regex
- max results
```

## 5. 文件监听

建议后端提供 WebSocket 或 SSE：

```txt
WS /api/workspaces/:workspaceId/watch
```

事件：

```ts
type FileWatchEvent =
  | { type: "created"; path: string; node?: DirectoryNode }
  | { type: "changed"; path: string; mtime?: number; version?: string; hash?: string }
  | { type: "deleted"; path: string }
  | { type: "renamed"; oldPath: string; newPath: string };
```

前端处理：

```txt
- created：刷新对应目录 query。
- changed：未打开只刷新 stat/tree；clean 打开文件提示 reload 或自动 reload；dirty 打开文件进入 compare / overwrite / reload / cancel 冲突流程。
- deleted：复用 EditorService.onPathDeleted；dirty Tab 保留内容并标记 deleted。
- renamed：只有同时拿到 oldPath/newPath 时才复用 EditorService.onPathRenamed。
- 如果底层 watcher 只能提供 delete + create，不猜测 rename，按 delete 和 create 分别处理。
```

保存冲突：

```txt
- 写文件必须携带 expectedModifiedAt / expectedSize；后续可升级为 version/hash。
- 409 file_write_conflict 是最终安全边界，前端 preflight 只能优化体验，不能替代后端校验。
- force/overwrite 只能由用户显式触发，不能在 dirty save 中默认启用。
```

## 6. 终端接口

WebSocket：

```txt
WS /api/workspaces/:workspaceId/terminal
```

客户端消息：

```ts
type TerminalClientMessage =
  | { type: "create"; cwd?: string; shell?: string }
  | { type: "input"; sessionId: string; data: string }
  | { type: "resize"; sessionId: string; cols: number; rows: number }
  | { type: "close"; sessionId: string };
```

服务端消息：

```ts
type TerminalServerMessage =
  | { type: "created"; sessionId: string; cwd: string; shell: string }
  | { type: "output"; sessionId: string; data: string }
  | { type: "exit"; sessionId: string; code: number | null }
  | { type: "disconnected"; sessionId: string; reason?: string }
  | { type: "error"; sessionId: string; message: string };
```

后端职责：

```txt
- 创建 PTY。
- cwd 限制在 workspace。
- shell 白名单。
- resize。
- input/output 转发。
- kill session。
- 断线清理或保活策略。
- 权限检查 canOpenTerminal。
- close/kill 必须真实终止 PTY 进程。
- session 超时清理。
- 不默认持久化完整终端输出。
```

M5 的后端只需要支持真实终端基础能力；M5.x 的 terminal split/group 是前端布局和 session 编排增强，不应要求后端理解布局树。后端关注 session lifecycle、input/output、resize、权限和安全边界。

## 7. 任务运行接口

后续 IDE 需要运行任务，不建议所有任务都通过裸终端。

任务接口：

```txt
POST /api/workspaces/:workspaceId/tasks/run
GET  /api/workspaces/:workspaceId/tasks/:taskId
POST /api/workspaces/:workspaceId/tasks/:taskId/cancel
WS   /api/workspaces/:workspaceId/tasks/:taskId/logs
```

任务模型：

```ts
type WorkspaceTask = {
  id: string;
  name: string;
  command: string;
  args?: string[];
  cwd: string;
  status: "pending" | "running" | "success" | "failed" | "cancelled";
  startedAt?: string;
  finishedAt?: string;
};
```

常见任务：

```txt
- npm run dev
- npm test
- npm run build
- pnpm install
- python script.py
- shell script
```

## 8. 权限模型

至少需要区分：

```txt
- canReadFile
- canWriteFile
- canCreateFile
- canDeleteFile
- canRenameFile
- canOpenTerminal
- canRunTask
- canUseGit
- canUseLsp
```

权限不足时：

```txt
- 前端按钮禁用。
- 后端仍必须拒绝。
- 错误信息明确。
```

## 9. 审计与日志

生产环境建议记录：

```txt
- 文件写入
- 文件删除
- 文件重命名
- 终端创建
- 任务运行
- Git 操作
```

审计字段：

```ts
type AuditLog = {
  userId: string;
  workspaceId: string;
  action: string;
  path?: string;
  command?: string;
  timestamp: string;
  success: boolean;
  error?: string;
};
```

注意：不建议默认记录完整文件内容和完整终端输出，避免隐私和安全问题。

## 10. 后端验收标准

```txt
- 所有文件操作都限制在 workspace root 内。
- ../、绝对路径、软链接逃逸被拒绝。
- 读文件返回 encoding、eol、mtime、version。
- 写文件支持 expectedModifiedAt / expectedSize 冲突检测；后续可升级为后端 version/hash。
- 大文件、二进制文件有明确错误码。
- 文件 watcher 能通知 create/change/delete/rename。
- 终端 session 能创建、输入、输出、resize、关闭。
- 权限不足时前后端都能正确处理。
```
