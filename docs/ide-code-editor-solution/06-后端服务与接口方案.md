# 后端服务与接口方案

## 1. 后端目标

前端的 Monaco 和 xterm.js 只负责展示和交互。

真正的能力需要后端提供：

```txt
- workspace 管理
- 文件读写
- 文件树
- 文件搜索
- 文件监听
- 权限校验
- 路径安全
- 终端 PTY
- 任务运行
- 后续 LSP 和 Git
```

## 2. Workspace 边界

所有接口都必须围绕 workspaceId。

```txt
workspaceId
→ 后端解析 rootPath
→ 所有 path 都必须限制在 rootPath 内
```

前端不传绝对路径。

正确：

```json
{
  "workspaceId": "demo",
  "path": "/src/App.tsx"
}
```

不推荐：

```json
{
  "path": "C:\\Users\\xxx\\project\\src\\App.tsx"
}
```

## 3. 路径安全

后端必须做 root guard。

需要防止：

```txt
- ../ 路径逃逸
- URL 编码绕过
- 软链接逃逸
- Windows 盘符路径绕过
- UNC 路径绕过
- 大小写路径绕过
- 删除 workspace root
```

伪代码：

```ts
function resolveWorkspacePath(workspaceRoot: string, userPath: string) {
  const resolved = path.resolve(workspaceRoot, "." + normalizeUserPath(userPath));
  const root = path.resolve(workspaceRoot);

  if (!isSubPath(resolved, root)) {
    throw new Error("Path escapes workspace root");
  }

  return resolved;
}
```

注意 Windows 下要处理大小写和盘符规范化。

## 4. 文件接口

### 获取目录树

```txt
GET /api/workspaces/:workspaceId/tree?path=/src
```

返回：

```ts
type DirectoryNode = {
  path: string;
  name: string;
  type: "file" | "folder";
  size?: number;
  mtime?: number;
  readonly?: boolean;
  hidden?: boolean;
  childrenLoaded?: boolean;
};
```

建议：

```txt
- 目录懒加载。
- 支持排序。
- 支持隐藏文件配置。
- 支持忽略 node_modules、.git 等大目录。
```

### 读取文件

```txt
GET /api/workspaces/:workspaceId/file?path=/src/App.tsx
```

返回：

```ts
type ReadFileResponse = {
  path: string;
  content: string;
  encoding: string;
  eol: "LF" | "CRLF";
  language?: string;
  size: number;
  mtime: number;
  version: string;
  readonly: boolean;
};
```

读取前校验：

```txt
- 是否存在
- 是否为文件
- 是否二进制
- 文件大小是否超过阈值
- 是否有权限
```

### 写入文件

```txt
PUT /api/workspaces/:workspaceId/file
```

请求：

```ts
type WriteFileRequest = {
  path: string;
  content: string;
  encoding?: string;
  eol?: "LF" | "CRLF";
  expectedVersion?: string;
};
```

返回：

```ts
type WriteFileResponse = {
  path: string;
  saved: true;
  mtime: number;
  version: string;
};
```

冲突处理：

```txt
如果 expectedVersion 与当前文件版本不一致：
- 返回 409 Conflict。
- 前端提示文件已被外部修改。
- 可选择比较差异、覆盖保存、重新加载。
```

### 新建文件

```txt
POST /api/workspaces/:workspaceId/file
```

请求：

```ts
type CreateFileRequest = {
  path: string;
  content?: string;
  overwrite?: boolean;
};
```

### 新建目录

```txt
POST /api/workspaces/:workspaceId/folder
```

请求：

```ts
type CreateFolderRequest = {
  path: string;
};
```

### 重命名

```txt
PATCH /api/workspaces/:workspaceId/path/rename
```

请求：

```ts
type RenamePathRequest = {
  oldPath: string;
  newPath: string;
  overwrite?: boolean;
};
```

### 删除

```txt
DELETE /api/workspaces/:workspaceId/path?path=/src/App.tsx
```

建议：

```txt
- 默认进入回收站或软删除。
- 危险删除需要二次确认。
- 不允许删除 workspace root。
```

### 文件搜索

```txt
GET /api/workspaces/:workspaceId/search?q=keyword&path=/src
```

支持：

```txt
- include glob
- exclude glob
- case sensitive
- regex
- max results
```

## 5. 文件监听

建议后端提供 WebSocket 或 SSE：

```txt
WS /api/workspaces/:workspaceId/watch
```

事件：

```ts
type FileWatchEvent =
  | { type: "created"; path: string; node: DirectoryNode }
  | { type: "changed"; path: string; mtime: number; version: string }
  | { type: "deleted"; path: string }
  | { type: "renamed"; oldPath: string; newPath: string };
```

前端处理：

```txt
- 刷新文件树节点。
- 打开的 clean 文件提示 reload 或自动 reload。
- 打开的 dirty 文件提示冲突。
```

## 6. 终端接口

WebSocket：

```txt
WS /api/workspaces/:workspaceId/terminal
```

客户端消息：

```ts
type TerminalClientMessage =
  | { type: "create"; cwd?: string; shell?: string }
  | { type: "input"; sessionId: string; data: string }
  | { type: "resize"; sessionId: string; cols: number; rows: number }
  | { type: "close"; sessionId: string };
```

服务端消息：

```ts
type TerminalServerMessage =
  | { type: "created"; sessionId: string; cwd: string; shell: string }
  | { type: "output"; sessionId: string; data: string }
  | { type: "exit"; sessionId: string; code: number | null }
  | { type: "error"; sessionId: string; message: string };
```

后端职责：

```txt
- 创建 PTY。
- cwd 限制在 workspace。
- shell 白名单。
- resize。
- input/output 转发。
- kill session。
- 断线清理或保活策略。
```

## 7. 任务运行接口

后续 IDE 需要运行任务，不建议所有任务都通过裸终端。

任务接口：

```txt
POST /api/workspaces/:workspaceId/tasks/run
GET  /api/workspaces/:workspaceId/tasks/:taskId
POST /api/workspaces/:workspaceId/tasks/:taskId/cancel
WS   /api/workspaces/:workspaceId/tasks/:taskId/logs
```

任务模型：

```ts
type WorkspaceTask = {
  id: string;
  name: string;
  command: string;
  args?: string[];
  cwd: string;
  status: "pending" | "running" | "success" | "failed" | "cancelled";
  startedAt?: string;
  finishedAt?: string;
};
```

常见任务：

```txt
- npm run dev
- npm test
- npm run build
- pnpm install
- python script.py
- shell script
```

## 8. 权限模型

至少需要区分：

```txt
- canReadFile
- canWriteFile
- canCreateFile
- canDeleteFile
- canRenameFile
- canOpenTerminal
- canRunTask
- canUseGit
- canUseLsp
```

权限不足时：

```txt
- 前端按钮禁用。
- 后端仍必须拒绝。
- 错误信息明确。
```

## 9. 审计与日志

生产环境建议记录：

```txt
- 文件写入
- 文件删除
- 文件重命名
- 终端创建
- 任务运行
- Git 操作
```

审计字段：

```ts
type AuditLog = {
  userId: string;
  workspaceId: string;
  action: string;
  path?: string;
  command?: string;
  timestamp: string;
  success: boolean;
  error?: string;
};
```

注意：不建议默认记录完整文件内容和完整终端输出，避免隐私和安全问题。

## 10. 后端验收标准

```txt
- 所有文件操作都限制在 workspace root 内。
- ../、绝对路径、软链接逃逸被拒绝。
- 读文件返回 encoding、eol、mtime、version。
- 写文件支持 expectedVersion 冲突检测。
- 大文件、二进制文件有明确错误码。
- 文件 watcher 能通知 create/change/delete/rename。
- 终端 session 能创建、输入、输出、resize、关闭。
- 权限不足时前后端都能正确处理。
```
