# 共享内核与总体架构

## 1. 总体目标

无论是文件管理器在线编辑器，还是独立 IDE 工作台，都不应该各自实现一套文件读取、保存、Tab、dirty 状态和 Monaco model 管理。

推荐抽象出一套共享内核。

```txt
共享内核目标：
- 文件读写统一
- 文件路径和权限校验统一
- Monaco model 生命周期统一
- dirty 状态统一
- 保存、另存、全部保存统一
- 文件重命名、删除、外部变更处理统一
- 后续 LSP、Diff、Problems 可复用
```

## 2. 分层架构


## 3. 核心服务

### WorkspaceService

负责 workspace 级上下文。

```ts
type Workspace = {
  id: string;
  name: string;
  rootPath: string;
  readonly?: boolean;
  runtime?: "local" | "container" | "remote";
};
```

职责：

```txt
- 获取当前 workspace
- 解析 workspace root
- 判断 workspace 是否只读
- 管理最近打开 workspace
- 管理 workspace 配置
```

### FileService

负责文件系统操作。

```ts
interface FileService {
  listDirectory(input: ListDirectoryInput): Promise<DirectoryNode[]>;
  readFile(input: ReadFileInput): Promise<FileContent>;
  writeFile(input: WriteFileInput): Promise<WriteFileResult>;
  createFile(input: CreateFileInput): Promise<FileNode>;
  createFolder(input: CreateFolderInput): Promise<FileNode>;
  renamePath(input: RenamePathInput): Promise<RenamePathResult>;
  deletePath(input: DeletePathInput): Promise<DeletePathResult>;
  stat(input: StatInput): Promise<FileStat>;
  search(input: SearchInput): Promise<SearchResult[]>;
}
```

原则：

```txt
- 所有路径必须通过后端校验。
- 前端只传 workspaceId + 相对路径。
- 不允许前端直接拼绝对路径。
- 读写文件必须返回 version 或 mtime，用于冲突检测。
```

### EditorService

负责打开、关闭、保存编辑器文件。

```ts
interface EditorService {
  openFile(path: string, options?: OpenFileOptions): Promise<EditorTab>;
  closeFile(tabId: string): Promise<void>;
  saveFile(tabId: string): Promise<void>;
  saveAll(): Promise<void>;
  revealFile(path: string): Promise<void>;
  markDirty(modelUri: string, dirty: boolean): void;
  getActiveEditor(): ActiveEditor | undefined;
}
```

职责：

```txt
- 打开文件
- 管理 Tab
- 管理 active editor
- 调用 MonacoModelService
- 调用 SaveService
- 处理关闭未保存确认
```

### MonacoModelService

负责 Monaco model 生命周期。

```ts
interface MonacoModelService {
  getOrCreateModel(file: FileContent): monaco.editor.ITextModel;
  getModelByPath(path: string): monaco.editor.ITextModel | undefined;
  disposeModel(path: string): void;
  updateModelPath(oldPath: string, newPath: string): void;
  saveViewState(tabId: string, viewState: unknown): void;
  restoreViewState(tabId: string): unknown;
}
```

关键原则：

```txt
- 一个文件对应一个 Monaco model。
- 内容由 Monaco model 承载。
- React state 不保存完整文本。
- model URI 使用稳定规则。
- M1.x 需要保存/恢复每个 Tab 的 viewState：光标、选区、滚动位置。
```

URI 示例：

```txt
file:///workspace/{workspaceId}/src/App.tsx
```

### DirtyStateService

负责修改状态。

```ts
type DirtyState = {
  path: string;
  dirty: boolean;
  modelVersion: number;
  lastSavedVersion: number;
  lastSavedMtime?: number;
};
```

职责：

```txt
- 标记文件是否修改
- 判断是否需要保存
- 关闭前确认
- 页面刷新前确认
- 批量保存时筛选 dirty 文件
- 保存 lastSaved mtime/version/hash 等冲突检测 token
- 外部修改时区分 clean reload 与 dirty conflict
```

### CommandService

负责所有命令统一分发。

```ts
type CommandId =
  | "file.open"
  | "file.save"
  | "file.saveAll"
  | "file.close"
  | "file.closeOthers"
  | "editor.find"
  | "editor.replace"
  | "editor.gotoLine"
  | "workbench.toggleTerminal"
  | "workbench.toggleSidebar"
  | "terminal.new"
  | "terminal.kill";
```

职责：

```txt
- 菜单点击
- 快捷键
- 工具栏按钮
- 右键菜单
- 命令面板
```

都应该走同一套命令，不要每个组件直接写业务逻辑。

## 5. 事件流

### 打开文件

```txt
Explorer/FileManager
→ CommandService.execute("file.open", { path })
→ EditorService.openFile(path)
→ FileService.readFile(path)
→ MonacoModelService.getOrCreateModel(file)
→ TabService.addOrActivateTab(path)
→ UI 激活编辑器
```

### 保存文件

```txt
Ctrl+S / 保存按钮
→ CommandService.execute("file.save")
→ EditorService.saveFile(activeTabId)
→ MonacoModelService.getModelByPath(path).getValue()
→ SaveService.writeFile(path, value, expectedVersion)
→ DirtyStateService.markClean(path)
→ StatusBar 更新
```

### 关闭文件

```txt
点击 Tab 关闭
→ EditorService.closeFile(tabId)
→ DirtyStateService 判断 dirty
→ 如果 dirty，弹出保存/不保存/取消
→ 如果最后一个引用关闭，MonacoModelService disposeModel
```

### 文件重命名

```txt
Explorer rename
→ FileService.renamePath(oldPath, newPath)
→ FileTree 更新
→ EditorService.onPathRenamed(oldPath, newPath)
→ Tab path/title 更新
→ Monaco model URI 迁移或重建
→ DirtyState 跟随迁移
```

### 文件删除

```txt
Explorer delete
→ 检查是否有打开的 dirty tab
→ 确认后 FileService.deletePath(path)
→ EditorService.onPathDeleted(path)
→ 关闭 Tab 或标记 deleted
```

### 外部文件变化

```txt
WatchService 收到 changed
→ 如果文件未打开，刷新文件树
→ 如果文件已打开且 clean，提示 reload 或自动 reload
→ 如果文件已打开且 dirty，提示 compare / overwrite / reload
```

## 6. 状态存储建议

可以使用项目现有状态管理方案。

如果没有现成方案，推荐：

```txt
Zustand:
- workbenchStore
- editorStore
- terminalStore
- layoutStore

React Query / TanStack Query:
- 文件树
- 文件 stat
- 搜索结果
- 后端 API 数据
```

不要把 Monaco 文件内容放进 store。

Store 只保存：

```txt
- 打开的 Tab
- activeTabId
- editorGroups
- dirty 状态
- layout 状态
- terminal sessions
- selected file
- expanded folders
```

## 8. 设计约束

必须遵守：

```txt
- 读写文件必须统一走 FileService。
- 打开文件必须统一走 EditorService。
- 保存必须统一走 SaveService。
- Monaco model 不由业务组件随意创建。
- Tab 状态不和文件树状态混在一起。
- IDE 布局状态不影响文件管理器编辑弹窗。
- 快捷键、菜单、工具栏统一走 CommandService。
```

这样才能同时支持：

```txt
- 文件管理器在线编辑器
- 独立 IDE
- 后续嵌入式代码片段编辑器
- 后续 Diff 查看器
- 后续 Markdown/JSON/YAML 专用编辑器
```
