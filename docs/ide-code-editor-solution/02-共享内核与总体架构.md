# 共享内核与总体架构

## 1. 总体目标

无论是文件管理器在线编辑器，还是独立 IDE 工作台，都不应该各自实现一套文件读取、保存、Tab、dirty 状态和 Monaco model 管理。

推荐抽象出一套共享内核。

```txt
共享内核目标：
- 文件读写统一
- 文件路径和权限校验统一
- Monaco model 生命周期统一
- dirty 状态统一
- 保存、另存、全部保存统一
- 文件重命名、删除、外部变更处理统一
- 后续 LSP、Diff、Problems 可复用
```

## 2. 分层架构

```txt
UI Layer
├─ File Manager Editor UI
└─ IDE Workbench UI

Workbench Layer
├─ Layout Manager
├─ Tab Manager
├─ Command Manager
├─ Shortcut Manager
└─ Panel Manager

Editor Core Layer
├─ EditorService
├─ MonacoModelService
├─ DirtyStateService
├─ SaveService
├─ LanguageService
└─ DiffService

Workspace / Explorer Layer
├─ FileService
├─ WorkspaceService
├─ ExplorerService
├─ ExplorerTreeState
├─ FileOperationService
├─ WatchService
├─ SearchService
└─ PermissionService

Backend Layer
├─ File APIs
├─ Terminal Gateway
├─ Task Runner
├─ LSP Gateway
├─ Git Service
└─ Workspace Runtime
```

## 3. 核心服务

### WorkspaceService

负责 workspace 级上下文。

```ts
type Workspace = {
  id: string;
  name: string;
  rootPath: string;
  readonly?: boolean;
  runtime?: "local" | "container" | "remote";
};
```

职责：

```txt
- 获取当前 workspace
- 解析 workspace root
- 判断 workspace 是否只读
- 管理最近打开 workspace
- 管理 workspace 配置
```

### FileService

负责文件系统操作。

```ts
interface FileService {
  listDirectory(input: ListDirectoryInput): Promise<DirectoryNode[]>;
  readFile(input: ReadFileInput): Promise<FileContent>;
  writeFile(input: WriteFileInput): Promise<WriteFileResult>;
  createFile(input: CreateFileInput): Promise<FileNode>;
  createFolder(input: CreateFolderInput): Promise<FileNode>;
  renamePath(input: RenamePathInput): Promise<RenamePathResult>;
  movePath(input: MovePathInput): Promise<MovePathResult>;
  deletePath(input: DeletePathInput): Promise<DeletePathResult>;
  stat(input: StatInput): Promise<FileStat>;
  search(input: SearchInput): Promise<SearchResult[]>;
}
```

原则：

```txt
- 所有路径必须通过后端校验。
- 前端只传 workspaceId + 相对路径。
- 不允许前端直接拼绝对路径。
- 读写文件必须返回 version 或 mtime，用于冲突检测。
```

### EditorService

负责打开、关闭、保存编辑器文件。

```ts
interface EditorService {
  openFile(path: string, options?: OpenFileOptions): Promise<EditorTab>;
  closeFile(tabId: string): Promise<void>;
  saveFile(tabId: string): Promise<void>;
  saveAll(): Promise<void>;
  revealFile(path: string): Promise<void>;
  markDirty(modelUri: string, dirty: boolean): void;
  onPathRenamed(input: { rootId: string; oldPath: string; newPath: string }): void;
  onPathDeleted(input: { rootId: string; path: string }): void;
  getActiveEditor(): ActiveEditor | undefined;
}
```

职责：

```txt
- 打开文件
- 管理 Tab
- 管理 active editor
- 调用 MonacoModelService
- 调用 SaveService
- 处理关闭未保存确认
- 处理重命名、移动、删除后打开中 Tab / Monaco model / dirty 状态同步
```

### MonacoModelService

负责 Monaco model 生命周期。

```ts
interface MonacoModelService {
  getOrCreateModel(file: FileContent): monaco.editor.ITextModel;
  getModelByPath(path: string): monaco.editor.ITextModel | undefined;
  disposeModel(path: string): void;
  updateModelPath(oldPath: string, newPath: string): void;
  saveViewState(tabId: string, viewState: unknown): void;
  restoreViewState(tabId: string): unknown;
}
```

关键原则：

```txt
- 一个文件对应一个 Monaco model。
- 内容由 Monaco model 承载。
- React state 不保存完整文本。
- model URI 使用稳定规则。
- M1.x 需要保存/恢复每个 Tab 的 viewState：光标、选区、滚动位置。
```

URI 示例：

```txt
file:///workspace/{workspaceId}/src/App.tsx
```

### DirtyStateService

负责修改状态。

```ts
type DirtyState = {
  path: string;
  dirty: boolean;
  modelVersion: number;
  lastSavedVersion: number;
  lastSavedMtime?: number;
};
```

职责：

```txt
- 标记文件是否修改
- 判断是否需要保存
- 关闭前确认
- 页面刷新前确认
- 批量保存时筛选 dirty 文件
- 保存 lastSaved mtime/version/hash 等冲突检测 token
- 外部修改时区分 clean reload 与 dirty conflict
```

### ExplorerService

负责文件树/列表导航与文件操作命令的共享契约。在线编辑器 Mini Explorer 和未来独立 IDE Explorer 都应复用这一层。

```ts
type ExplorerLocation = {
  rootId: string;
  path: string;
};

interface ExplorerService {
  list(input: ExplorerLocation): Promise<ExplorerNode[]>;
  refresh(input: ExplorerLocation): Promise<void>;
  parent(input: ExplorerLocation): ExplorerLocation;
  createFile(input: ExplorerLocation, name: string): Promise<ExplorerNode>;
  createDirectory(input: ExplorerLocation, name: string): Promise<ExplorerNode>;
  rename(input: ExplorerLocation, nextName: string): Promise<ExplorerNode>;
  remove(input: ExplorerLocation[]): Promise<void>;
  copy(input: ExplorerLocation, destination: ExplorerLocation): Promise<void>;
  move(input: ExplorerLocation, destination: ExplorerLocation): Promise<void>;
}
```

职责：

```txt
- 统一目录读取、刷新、父级导航。
- 统一文件树节点模型、排序、过滤、图标和权限判断。
- 统一新建/重命名/删除/复制/移动命令。
- 向 EditorService 发出 openFile 意图，不直接操作 Monaco model。
- 在线编辑器 Mini Explorer 使用轻量 UI；独立 IDE Explorer 使用 Workbench SideBar UI。
```

### CommandService

负责所有命令统一分发。

```ts
type CommandId =
  | "file.open"
  | "file.save"
  | "file.saveAll"
  | "file.close"
  | "file.closeOthers"
  | "explorer.toggle"
  | "explorer.refresh"
  | "explorer.newFile"
  | "explorer.newDirectory"
  | "explorer.rename"
  | "explorer.delete"
  | "explorer.copy"
  | "explorer.move"
  | "editor.find"
  | "editor.replace"
  | "editor.gotoLine"
  | "editor.splitRight"
  | "editor.splitDown"
  | "editor.closeGroup"
  | "workbench.togglePanel"
  | "workbench.toggleSidebar"
  | "workbench.resetLayout"
  | "terminal.new"
  | "terminal.kill";
```

职责：

```txt
- 菜单点击
- 快捷键
- 工具栏按钮
- 右键菜单
- 命令面板
```

都应该走同一套命令，不要每个组件直接写业务逻辑。

统一入口规则：

```txt
- FileManager、Mini Explorer、IDE Explorer、EditorArea、PanelArea 不直接互相调用。
- UI 组件只发起 CommandService.execute(commandId, payload)。
- Command handler 负责权限判断、dirty 确认、保存冲突处理、路径同步和布局状态更新。
- 快捷键、右键菜单、工具栏按钮、Tab 菜单、Command Palette 必须调用同一个 commandId。
- M3/M4 只需要轻量 command registry，不做插件系统、不做第三方命令扩展。
- 后续 IDE Command Palette 可以直接读取 command registry 的 id、title、category、快捷键和 enablement。
```

建议命令元数据：

```ts
type CommandDescriptor = {
  id: CommandId;
  title: string;
  category: "file" | "explorer" | "editor" | "workbench" | "terminal";
  keybinding?: string;
  when?: string;
  enablement?: (ctx: CommandContext) => boolean;
};
```

## 5. 事件流

### 打开文件

```txt
Explorer/FileManager
→ CommandService.execute("file.open", { path })
→ EditorService.openFile(path)
→ FileService.readFile(path)
→ MonacoModelService.getOrCreateModel(file)
→ TabService.addOrActivateTab(path)
→ UI 激活编辑器
```

### 保存文件

```txt
Ctrl+S / 保存按钮
→ CommandService.execute("file.save")
→ EditorService.saveFile(activeTabId)
→ MonacoModelService.getModelByPath(path).getValue()
→ SaveService.writeFile(path, value, expectedVersion)
→ DirtyStateService.markClean(path)
→ StatusBar 更新
```

### 关闭文件

```txt
点击 Tab 关闭
→ EditorService.closeFile(tabId)
→ DirtyStateService 判断 dirty
→ 如果 dirty，弹出保存/不保存/取消
→ 如果最后一个引用关闭，MonacoModelService disposeModel
```

### 文件重命名 / 移动

```txt
FileManager / Mini Explorer / IDE Explorer rename or move
→ FileService.renamePath 或 movePath(oldPath, newPath)
→ Explorer query invalidate / refresh
→ EditorService.onPathRenamed({ rootId, oldPath, newPath })
→ 已打开 Tab path/title 更新
→ Monaco model URI 迁移或重建，但内容和 dirty 状态保留
→ DirtyState / SaveService expected token 跟随新 path 迁移
```

规则：

```txt
- rename 和 move 对编辑器来说都是 path 迁移。
- 未 dirty 的已打开文件自动更新 path/title/model key。
- dirty 的已打开文件也更新 path/title/model key，未保存内容继续保留，不能因为改名丢失。
- 同一 rootId + oldPath 的所有产品壳层 Tab 都要同步：文件管理器在线编辑器、Mini Explorer、独立 IDE EditorGroup。
- 同一文件在多个 IDE EditorGroup 打开时，所有 Tab 都更新到新 path，仍共享同一个底层 model/save 状态。
```

### 文件删除

```txt
FileManager / Mini Explorer / IDE Explorer delete
→ 如果命中打开中的 dirty tab，先提示并要求明确选择
→ FileService.deletePath(path)
→ Explorer query invalidate / refresh
→ EditorService.onPathDeleted({ rootId, path })
→ clean Tab 可关闭或标记 deleted
→ dirty Tab 必须保留内容并标记 deleted，提供另存为/重新创建/放弃修改
```

规则：

```txt
- 删除未 dirty 的已打开文件：可以自动关闭 Tab，或保留 deleted 状态让用户知道文件已不存在；产品壳层要保持一致。
- 删除 dirty 的已打开文件：不能静默关闭，不能丢弃 Monaco model 内容。
- deleted Tab 保存时不能静默写回原路径，必须提示“文件已删除”，让用户选择另存为、重新创建或放弃。
- 目录删除如果覆盖多个已打开文件，按子路径批量应用同一规则，并在确认文案中显示影响数量。
```

### 外部文件变化 / 保存冲突

WatchService 事件必须归一成少数稳定事件，不让在线编辑器、Mini Explorer 和独立 IDE 各自处理一套冲突逻辑。

```ts
type NormalizedFileWatchEvent =
  | { type: "created"; rootId: string; path: string }
  | { type: "changed"; rootId: string; path: string; mtime?: number; version?: string; hash?: string }
  | { type: "deleted"; rootId: string; path: string }
  | { type: "renamed"; rootId: string; oldPath: string; newPath: string };
```

处理规则：

```txt
WatchService 收到 created
→ 刷新对应目录 query，不影响已打开编辑内容

WatchService 收到 changed
→ 如果文件未打开，只刷新文件树/stat cache
→ 如果文件已打开且 clean，提示 reload 或轻提示后自动 reload，并更新 lastSaved token
→ 如果文件已打开且 dirty，进入冲突状态，提示 compare / overwrite / reload / cancel，不能静默覆盖

WatchService 收到 deleted
→ 复用 EditorService.onPathDeleted({ rootId, path })
→ clean Tab 可关闭或标记 deleted
→ dirty Tab 保留内容并标记 deleted，提示另存为/重新创建/放弃修改

WatchService 收到 renamed 且同时有 oldPath/newPath
→ 复用 EditorService.onPathRenamed({ rootId, oldPath, newPath })
→ Tab path/title/model key 与 dirty/save token 迁移

WatchService 只能收到 delete + create，无法可靠证明 rename
→ 不猜测 rename
→ 先按 deleted 处理旧路径，created 只刷新目录
```

保存规则：

```txt
- 保存必须带 expected version/mtime/hash 或当前后端支持的 expectedModifiedAt/expectedSize。
- 后端返回 409 file_write_conflict 时，前端进入同一套 compare / overwrite / reload / cancel 流程。
- overwrite 必须是用户显式选择；不能由 dirty save 自动强制覆盖。
- reload 会丢弃本地 dirty 内容时必须二次确认。
```

## 6. 状态存储建议

可以使用项目现有状态管理方案。

如果没有现成方案，推荐：

```txt
Zustand:
- workbenchStore
- editorStore
- terminalStore
- layoutStore

React Query / TanStack Query:
- 文件树
- 文件 stat
- 搜索结果
- 后端 API 数据
```

不要把 Monaco 文件内容放进 store。

Store 只保存：

```txt
- 打开的 Tab
- activeTabId
- editorGroups
- dirty 状态
- layout 状态和 layoutVersion
- selected file
- expanded folders
- explorer collapsed/open state
- mini explorer drawer state on small screens
```

持久化分层：

```txt
user 级：主题、字体、快捷键、通用布局偏好。
workspace 级：打开文件、编辑组、Explorer 展开/选中、SideBar/Panel/viewPlacements。
session 级：focus、hover、临时 selection、拖拽中间态。
```

M4 不持久化真实 terminal session、terminal scrollback、LSP/Git/Debug runtime 状态。布局恢复失败必须 fallback 默认布局；`workbench.resetLayout` 只清 workspace 布局，不丢 dirty 编辑内容。

## 8. 设计约束

必须遵守：

```txt
- 读写文件必须统一走 FileService。
- 打开文件必须统一走 EditorService。
- 保存必须统一走 SaveService。
- Monaco model 不由业务组件随意创建。
- Tab 状态不和文件树状态混在一起。
- Explorer Core 可共享目录和文件操作，但 Mini Explorer/IDE Explorer 的布局状态必须隔离。
- IDE 布局状态不影响文件管理器编辑弹窗。
- 快捷键、菜单、工具栏统一走 CommandService。
```

这样才能同时支持：

```txt
- 文件管理器在线编辑器
- 独立 IDE
- 后续嵌入式代码片段编辑器
- 后续 Diff 查看器
- 后续 Markdown/JSON/YAML 专用编辑器
```
