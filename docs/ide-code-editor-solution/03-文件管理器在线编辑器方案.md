# 文件管理器在线编辑器方案

## 1. 产品定位

文件管理器在线编辑器是文件管理器中的快速编辑能力。

参考形态类似：

```txt
文件管理器
→ 双击文件
→ 打开在线文本编辑器
→ 多 Tab 编辑
→ 保存/全部保存/搜索/替换/跳转行
```

它不是完整 IDE，不负责终端、Git、调试、复杂布局。

## 2. 推荐入口

文件管理器中可以提供以下入口：

```txt
- 双击可编辑文本文件
- 右键菜单：编辑
- 文件列表操作列：编辑
- 快捷键 Enter / Ctrl+Enter 打开
- 顶部按钮：打开编辑器
```

可编辑文件类型：

```txt
- .txt
- .md
- .json
- .yaml / .yml
- .xml
- .html
- .css
- .scss
- .less
- .js / .jsx
- .ts / .tsx
- .vue
- .py
- .java
- .go
- .rs
- .php
- .sh
- .bat / .cmd
- .ps1
- .conf
- .env
- .ini
- .log
```

不可直接编辑文件：

```txt
- 图片
- 视频
- 音频
- 压缩包
- 二进制文件
- 超大文件
- 无权限文件
```

不可编辑时应提示原因，不要强行用文本打开。

## 3. 容器形态

根据项目 UI 可选择：

```txt
方案 A：普通弹窗
- 适合快速编辑小文件。
- 实现成本低。

方案 B：全屏弹窗
- 接近宝塔在线编辑器体验。
- 更适合多 Tab 和搜索替换。

方案 C：右侧抽屉
- 适合边看文件列表边编辑。
- 编辑区域较小，不适合复杂代码。

方案 D：独立路由
- /file-editor?workspaceId=xxx&path=xxx
- 适合大文件编辑和可分享链接。
```

推荐第一版：

```txt
全屏弹窗或独立路由。
```

## 5. 必须功能

### 打开文件

行为：

```txt
- 双击文件打开。
- 如果已经打开，则激活已有 Tab。
- 如果未打开，则读取文件并创建 Tab。
- 打开失败时显示错误。
- 无权限时显示只读或禁止打开。
```

### 多 Tab

能力：

```txt
- 一个文件一个 Tab。
- Tab 显示文件名。
- dirty 文件显示修改标记。
- 支持关闭单个 Tab。
- 支持关闭其他 Tab。
- 支持关闭全部。
- 支持关闭已保存。
- 点击 Tab 切换文件。
- 切换时保留光标、选区、滚动位置。
```

Tab 状态：

```ts
type FileEditorTab = {
  id: string;
  path: string;
  title: string;
  language: string;
  dirty: boolean;
  readonly: boolean;
  active: boolean;
  viewState?: unknown;
};
```

### 保存

保存能力：

```txt
- 保存当前文件。
- 保存全部文件。
- Ctrl+S 保存当前文件。
- Ctrl+Shift+S 保存全部文件。
- 保存成功后 dirty 标记消失。
- 保存失败保留 dirty 标记。
```

保存前校验：

```txt
- 文件是否仍然存在。
- 文件是否只读。
- 文件是否被外部修改。
- 用户是否有写权限。
```

### 关闭确认

关闭场景：

```txt
- 关闭 Tab。
- 关闭编辑器弹窗。
- 刷新浏览器。
- 跳转离开页面。
```

如果存在未保存文件，提示：

```txt
- 保存
- 不保存
- 取消
```

如果多个文件未保存，提示数量并支持全部保存。

### 搜索与替换

需要支持：

```txt
- 当前文件搜索
- 当前文件替换
- 区分大小写
- 全词匹配
- 正则表达式
- 搜索结果计数
- 上一个/下一个
```

第一版可以直接使用 Monaco 内置 find/replace。

### 跳转行

需要支持：

```txt
- 输入行号跳转
- 支持行:列
- 输入非法时提示
```

例如：

```txt
12
12:8
```

### 字体大小

需要支持：

```txt
- 增大字体
- 减小字体
- 恢复默认
- 配置持久化到用户偏好
```

### 主题

需要支持：

```txt
- 跟随系统/项目主题
- 浅色
- 深色
- 高对比度可后续扩展
```

注意：本方案不规定具体配色。

### 状态栏

状态栏建议字段：

```txt
- 文件路径
- 当前行列
- 换行符：LF / CRLF
- 缩进：Spaces 2 / Spaces 4 / Tab Size 4
- 编码：UTF-8 / GBK / ASCII 等
- 语言：JavaScript / Shell / Markdown 等
- 保存状态：已保存 / 未保存 / 保存中 / 保存失败
```

## 6. 建议功能

### 重新加载

```txt
用户点击刷新当前文件
→ 如果 dirty，提示是否放弃修改
→ 从后端重新读取文件
→ 更新 Monaco model
```

### 只读模式

触发条件：

```txt
- 无写权限
- 文件被锁定
- workspace 只读
- 文件来自历史版本
```

行为：

```txt
- Monaco 设置 readOnly
- 保存按钮禁用
- 状态栏显示只读
```

### 外部修改提示

当 watcher 检测文件被外部修改：

```txt
如果当前 Tab 未修改：
- 提示文件已变化，是否重新加载。

如果当前 Tab 已修改：
- 提示文件已被外部修改。
- 选项：比较差异 / 覆盖保存 / 重新加载 / 取消。
```

### 大文件保护

建议阈值：

```txt
- 小于 1 MB：正常 Monaco 编辑。
- 1 MB - 5 MB：提示大文件，仍可编辑。
- 5 MB - 20 MB：默认只读打开，可强制编辑。
- 大于 20 MB：不建议 Monaco 编辑，使用只读查看或下载。
```

阈值可配置。

### 编码处理

第一版建议：

```txt
- 默认 UTF-8。
- 后端识别常见编码。
- 状态栏显示编码。
- 保存时沿用原编码。
```

后续可支持：

```txt
- 以指定编码重新打开。
- 转换编码保存。
```

## 7. 不建议放入第一版

文件管理器在线编辑器不建议第一版做：

```txt
- 终端
- Git
- 调试
- LSP 全量接入
- 多编辑组拆分
- 自定义 docking layout
- 插件系统
```

这些应该放到独立 IDE 工作台。

## 9. 交互细节

### 双击打开

```txt
if file.type !== "file":
  expand folder
else if isBinary(file):
  preview/download
else if file.size > maxEditableSize:
  show large file warning
else:
  openFileEditor(file.path)
```

### Tab 关闭

```txt
if tab.dirty:
  show confirm
  - save
  - discard
  - cancel
else:
  close tab
```

### 弹窗关闭

```txt
dirtyTabs = tabs.filter(tab => tab.dirty)
if dirtyTabs.length > 0:
  show confirm with count
else:
  close editor
```

### 文件被删除

```txt
如果打开中的文件被删除：
- Tab 显示 deleted 状态。
- 保存时提示文件不存在。
- 可提供另存为或重新创建。
```

## 10. 验收标准

第一版验收：

```txt
- 能从文件管理器打开文本文件。
- 同一个文件重复打开不会产生重复 Tab。
- 能同时打开多个文件。
- 能编辑并保存当前文件。
- 能全部保存。
- 未保存文件关闭时有确认。
- Tab 切换后光标和滚动位置能恢复。
- 文件路径、语言、行列、编码、换行符能显示。
- 只读文件不能保存。
- 文件不存在、权限不足、大文件、二进制文件有明确提示。
```
