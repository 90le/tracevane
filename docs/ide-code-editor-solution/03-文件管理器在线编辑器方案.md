# 文件管理器在线编辑器方案

## 1. 产品定位

文件管理器在线编辑器是文件管理器中的快速编辑能力。

参考形态类似：

```txt
文件管理器
→ 双击文件
→ 打开在线文本编辑器
→ 多 Tab 编辑
→ 保存/全部保存/搜索/替换/跳转行
```

它不是完整 IDE，不负责终端、Git、调试、复杂布局。

## 1.1 当前落地状态与 M1.x 方向

M1 已完成并验证了文件管理器在线编辑器的基础形态：

```txt
- 文件管理器 text-like 文件双击进入 Online Editor
- Monaco 多 Tab 编辑
- 同文件重复打开激活已有 Tab
- 保存当前 / 保存全部
- dirty 标记、关闭保护、beforeunload 保护
- 最小化驻留 dock 与恢复
- 查找、替换、跳转行、字号、主题入口
- 路径 / language / cursor / save / read-only 状态栏
- 大文件/截断只读、缺失文件、保存失败保留 dirty 等边界状态
```

M1.x 继续增强文件管理器在线编辑器，但仍不进入独立 IDE：

```txt
- [M1.x.1 已实现] 窗口最大化 / 还原 / 最小化 / 关闭
- [M1.x.1 已实现] Tab 关闭其他、关闭全部、关闭已保存
- [M1.x.1 已实现] 多 Tab 自适应缩小，达到最小宽度后横向滚动
- [M1.x.1 已实现] Tab 切换保留光标、选区、滚动 viewState
- [M1.x.2 已实现] 外部修改冲突检测与 lightweight compare / reload / overwrite / cancel；Monaco Diff 作为后续视觉增强
- [M1.x.3 已实现] 关闭确认升级为 保存 / 不保存 / 取消
- [M1.x.3 已实现] 重新读取当前文件；dirty reload 支持保存后重读 / 不保存重读 / 取消
- [M1.x.4 已实现] 状态栏展示换行符、缩进、编码、文件大小、权限、mtime、只读原因
- [M1.x.5 已实现] 搜索替换选项显式化：上/下一个、大小写、全词、正则，复用 Monaco 内置查找框；后续 Monaco-first cleanup 会删除重复外围控件，只保留薄入口
- [M1.x.6 已实现] 主题能力显式化：auto / light / dark；字体大小和主题偏好持久化
- [M1.x.7 已实现] 编辑入口补齐：右键“编辑”、行内编辑按钮、选中文件工具栏“打开编辑器”、Ctrl/Cmd+Enter
- light / dark / auto 主题选择和偏好持久化
- 右键菜单、操作列、顶部按钮等编辑入口
```

详细执行路线见 `m1x-execution-plan.md`。

## 1.2 Monaco-first 调整方向

M1.x 已补齐文件管理器在线编辑器的安全和外壳能力。后续方向应切换为 **Monaco-first cleanup**：

```txt
Monaco 负责：搜索/替换、多光标、撤销重做、折叠、context menu、minimap、word wrap、suggest/hover、快捷键、Diff Editor 等编辑器原生能力。
Tracevane 负责：多 Tab、文件读取/保存、dirty、Save All、reload、外部修改冲突、expected-token 409、最小化驻留、文件元数据状态栏、入口和大文件保护。
```

因此，已经实现的外围搜索按钮可作为过渡能力；下一步应按 `10-monaco-first-online-editor-strategy.md` 简化为 Find / Replace / Goto 等薄入口，让 Monaco 自己展示查找框、结果数、大小写/全词/正则、上/下一个等能力。


## 2. 推荐入口

文件管理器中可以提供以下入口：

```txt
- 双击可编辑文本文件
- 右键菜单：编辑
- 文件列表操作列：编辑
- 快捷键 Enter / Ctrl+Enter 打开
- 顶部按钮：打开编辑器
```

可编辑文件类型：

```txt
- .txt
- .md
- .json
- .yaml / .yml
- .xml
- .html
- .css
- .scss
- .less
- .js / .jsx
- .ts / .tsx
- .vue
- .py
- .java
- .go
- .rs
- .php
- .sh
- .bat / .cmd
- .ps1
- .conf
- .env
- .ini
- .log
```

不可直接编辑文件：

```txt
- 图片
- 视频
- 音频
- 压缩包
- 二进制文件
- 超大文件
- 无权限文件
```

不可编辑时应提示原因，不要强行用文本打开。

## 3. 容器形态

根据项目 UI 可选择：

```txt
方案 A：普通弹窗
- 适合快速编辑小文件。
- 实现成本低。

方案 B：全屏弹窗
- 接近宝塔在线编辑器体验。
- 更适合多 Tab 和搜索替换。

方案 C：右侧抽屉
- 适合边看文件列表边编辑。
- 编辑区域较小，不适合复杂代码。

方案 D：独立路由
- /file-editor?workspaceId=xxx&path=xxx
- 适合大文件编辑和可分享链接。
```

推荐第一版：

```txt
全屏弹窗或独立路由。
```

## 3.1 窗口控制

文件管理器在线编辑器应具备清晰的窗口控制：

```txt
- 最小化：隐藏编辑器窗口，保留 tabs 与 dirty 状态，显示驻留 dock。
- 恢复：从驻留 dock 恢复编辑器窗口。
- 最大化：占满可用视口，适合长时间编辑。
- 还原：从最大化回到常规弹窗尺寸。
- 关闭：若存在 dirty 文件，进入保存/放弃/取消流程。
```

窗口控制不等于独立 IDE 布局；M1.x 不引入 Dockview、多编辑组或底部面板。

当前 M1.x.1–M1.x.6 实现说明：最大化/还原通过在线编辑器窗口 mode 切换完成；最小化仍驻留在文件管理器右下角 dock；关闭 dirty tab / 关闭全部使用应用内“保存 / 不保存 / 取消”确认，重新读取当前 dirty 文件使用“保存后重新读取 / 不保存并重新读取 / 取消”确认；状态栏显示文件元数据；查找控制复用 Monaco；主题模式和字体大小偏好持久化；文本文件可从右键、行操作、选中工具栏和键盘快捷键进入在线编辑器。

## 5. 必须功能

### 打开文件

行为：

```txt
- 双击文件打开。
- 如果已经打开，则激活已有 Tab。
- 如果未打开，则读取文件并创建 Tab。
- 打开失败时显示错误。
- 无权限时显示只读或禁止打开。
```

### 多 Tab

能力：

```txt
- 一个文件一个 Tab。
- Tab 显示文件名。
- dirty 文件显示修改标记。
- 支持关闭单个 Tab。
- 支持关闭其他 Tab。
- 支持关闭全部。
- 支持关闭已保存。
- 点击 Tab 切换文件。
- 切换时保留光标、选区、滚动位置。
```

Tab 状态：

```ts
type FileEditorTab = {
  id: string;
  path: string;
  title: string;
  language: string;
  dirty: boolean;
  readonly: boolean;
  active: boolean;
  viewState?: unknown;
};
```

### 多 Tab 自适应与溢出

M1.x 的 Tab 行为建议：

```txt
- 不再用硬性 8 个标签上限作为产品能力边界。
- 打开的标签较少时，每个 Tab 保持可读宽度。
- 标签增多时，Tab 可自适应缩小到最小宽度。
- 达到最小宽度后，Tab 顶栏进入横向滚动。
- 任何情况下都不能为了容量限制静默丢弃 dirty Tab。
- 如果后续仍设置性能保护上限，必须先阻止打开并解释原因，不能自动关闭用户文件。
```

当前 M1.x.1 已落地：在线编辑器不再用 8 个 Tab 作为硬容量边界；Tab strip 使用最小宽度 + 横向滚动承载多文件，smoke 覆盖 16 个打开标签且不静默丢弃。

### 保存

保存能力：

```txt
- 保存当前文件。
- 保存全部文件。
- Ctrl+S 保存当前文件。
- Ctrl+Shift+S 保存全部文件。
- 保存成功后 dirty 标记消失。
- 保存失败保留 dirty 标记。
```

保存前校验：

```txt
- 文件是否仍然存在。
- 文件是否只读。
- 文件是否被外部修改。
- 用户是否有写权限。
```

### 关闭确认

关闭场景：

```txt
- 关闭 Tab。
- 关闭编辑器弹窗。
- 刷新浏览器。
- 跳转离开页面。
```

如果存在未保存文件，提示：

```txt
- 保存
- 不保存
- 取消
```

如果多个文件未保存，提示数量并支持全部保存。

### 搜索与替换

需要支持：

```txt
- 当前文件搜索
- 当前文件替换
- 区分大小写
- 全词匹配
- 正则表达式
- 搜索结果计数
- 上一个/下一个
```

第一版可以直接使用 Monaco 内置 find/replace。

### 跳转行

需要支持：

```txt
- 输入行号跳转
- 支持行:列
- 输入非法时提示
```

例如：

```txt
12
12:8
```

### 字体大小

需要支持：

```txt
- 增大字体
- 减小字体
- 恢复默认
- 配置持久化到用户偏好
```

### 主题

需要支持：

```txt
- 跟随系统/项目主题
- 浅色
- 深色
- 高对比度可后续扩展
```

注意：本方案不规定具体配色。

### 状态栏

状态栏建议字段：

```txt
- 文件路径
- 当前行列
- 换行符：LF / CRLF
- 缩进：Spaces 2 / Spaces 4 / Tab Size 4
- 编码：UTF-8 / GBK / ASCII 等
- 语言：JavaScript / Shell / Markdown 等
- 保存状态：已保存 / 未保存 / 保存中 / 保存失败
```

## 6. 建议功能

### 重新加载

```txt
用户点击刷新当前文件
→ 如果 dirty，提示是否放弃修改
→ 从后端重新读取文件
→ 更新 Monaco model
```

### 只读模式

触发条件：

```txt
- 无写权限
- 文件被锁定
- workspace 只读
- 文件来自历史版本
```

行为：

```txt
- Monaco 设置 readOnly
- 保存按钮禁用
- 状态栏显示只读
```

### 外部修改提示

当 watcher 检测文件被外部修改：

```txt
如果当前 Tab 未修改：
- 提示文件已变化，是否重新加载。

如果当前 Tab 已修改：
- 提示文件已被外部修改。
- 选项：比较差异 / 覆盖保存 / 重新加载 / 取消。
```

### 大文件保护

建议阈值：

```txt
- 小于 1 MB：正常 Monaco 编辑。
- 1 MB - 5 MB：提示大文件，仍可编辑。
- 5 MB - 20 MB：默认只读打开，可强制编辑。
- 大于 20 MB：不建议 Monaco 编辑，使用只读查看或下载。
```

阈值可配置。

### 编码处理

第一版建议：

```txt
- 默认 UTF-8。
- 后端识别常见编码。
- 状态栏显示编码。
- 保存时沿用原编码。
```

后续可支持：

```txt
- 以指定编码重新打开。
- 转换编码保存。
```

## 7. 不建议放入第一版

文件管理器在线编辑器不建议第一版做：

```txt
- 终端
- Git
- 调试
- LSP 全量接入
- 多编辑组拆分
- 自定义 docking layout
- 插件系统
```

这些应该放到独立 IDE 工作台。

## 9. 交互细节

### 双击打开

```txt
if file.type !== "file":
  expand folder
else if isBinary(file):
  preview/download
else if file.size > maxEditableSize:
  show large file warning
else:
  openFileEditor(file.path)
```

### Tab 关闭

```txt
if tab.dirty:
  show confirm
  - save
  - discard
  - cancel
else:
  close tab
```

### 弹窗关闭

```txt
dirtyTabs = tabs.filter(tab => tab.dirty)
if dirtyTabs.length > 0:
  show confirm with count
else:
  close editor
```

### 文件被删除

```txt
如果打开中的文件被删除：
- Tab 显示 deleted 状态。
- 保存时提示文件不存在。
- 可提供另存为或重新创建。
```

## 10. 验收标准

第一版验收：

```txt
- 能从文件管理器打开文本文件。
- 同一个文件重复打开不会产生重复 Tab。
- 能同时打开多个文件。
- 能编辑并保存当前文件。
- 能全部保存。
- 未保存文件关闭时有确认。
- Tab 切换后光标和滚动位置能恢复。
- 文件路径、语言、行列、编码、换行符能显示。
- 只读文件不能保存。
- 文件不存在、权限不足、大文件、二进制文件有明确提示。
```
