# OpenClaw Studio System Architecture

> 状态：Active
> 更新：2026-06-12

## 1. 项目定位

OpenClaw Studio 是 OpenClaw 的本地插件式管理工作台。它提供 Vue 前端、Node API、独立 daemon 和 typed config/store，用于管理 OpenClaw、Studio Gateway、Channel Connectors 和 Chat。

## 2. 顶层结构

```text
apps/api/        Studio API 与 daemon 入口
apps/web-vue/    Vue 前端工作台
types/           前后端共享类型
docs/            当前目标、合同、专项追踪
resources/       打包资源
tests/           系统与 smoke 测试
scripts/         构建、dev、smoke 辅助脚本
```

旧 `resources/codex-stack`、`features/codex-stack`、`modules/codex-stack` 不再是生产方向。

## 3. 后端模块

| 模块 | 职责 |
| --- | --- |
| `chat` | Studio typed chat BFF、history、send、stream、upload、runtime adapter |
| `studio-gateway` / model gateway | provider registry、secret、daemon lifecycle、adapter、route、usage、models |
| `channel-connectors` | 原生 IM daemon、Feishu/Octo transport、Agent runner、文件 staging、进度渲染 |
| `config` | OpenClaw/Studio 配置读写 |
| `agents` | Agent 管理、docs、bindings、sessions |
| `channels` | OpenClaw channels/accounts/bindings 管理 |
| `skills` | 技能市场、安装、启停、配置 |
| `terminal` | PTY/WebSocket、handoff、CLI 检查 |
| `system` | health、diagnostics、events、upgrade、runtime summary |
| `cron` | 定时任务和运行记录 |

## 4. 前端模块

前端按领域组织在 `apps/web-vue/src/features/<domain>`：

- `chat`
- `studio-gateway` / Gateway 管理面
- `channel-connectors`
- `config`
- `agents`
- `channels`
- `skills`
- `terminal`
- `system`
- `cron`

原则：

- 不恢复 Codex Stack 页面。
- Gateway 和 Channel Connectors 分页管理，不堆在同一页面。
- 前端只消费 Studio typed API，不直接消费 Gateway raw protocol。

## 5. Runtime / Daemon

| Daemon | 目标 |
| --- | --- |
| Studio Gateway daemon | 模型协议转换、provider routing、loopback endpoint、统一 Gateway key |
| Channel Connectors daemon | IM 长连接、私聊消息、Agent runner、文件收发、进度/命令 |
| OpenClaw recovery daemon | OpenClaw 运行态自愈，独立于 Studio Gateway / Channel Connectors |

守护策略：

- Linux 使用 `systemd --user`。
- macOS/Windows 后续使用 launchd / scheduled task 或 service。
- Studio API/UI 崩溃不应影响 Gateway daemon 和 Channel daemon 对外服务。

## 6. 数据与密钥

- 上游模型 key 写入 Studio secret store，不写入客户端配置。
- Gateway key 是本地统一 key，可编辑/轮换，不透传上游。
- Channel 平台 token、tenant token、附件、runtime、日志都在 Channel daemon data dir 内管理。
- 文档、测试和 fixture 不写真实凭据。

## 7. 测试基线

常用验证：

- `npm run typecheck:api`
- `npm run build:api`
- `npm run typecheck:web`
- `node --test tests/system/channel-connectors-service.test.mjs`
- 真实 provider / CLI / IM smoke 按变更范围选择运行。

文档-only 变更至少运行：

- `git diff --check`
- stale term 检查
