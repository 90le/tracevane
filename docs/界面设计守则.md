# OpenClaw Studio 界面设计守则

> 状态：Active
> 更新：2026-06-01

本文档是 `DESIGN.md` 的中文执行摘要。具体 token、组件、布局和 DuoYuan 参考细节以根目录 `DESIGN.md` 为准。

## 1. 产品气质

Studio 是本地运维控制台，不是营销页、门户页或纯展示 dashboard。界面应优先表现为：

- 稳定
- 清晰
- 可扫描
- 可恢复
- 对高风险操作有明确边界

视觉方向是 **DuoYuan Studio Ops**：

- light mode：接近白色的清爽控制台，不刺眼
- dark mode：graphite console，而不是蓝灰玻璃
- mint/teal 作为主操作识别
- peach/amber/red/green 只承担语义状态，不做装饰色泛滥

## 2. Shell 原则

- 左侧 rail 是唯一主导航
- topbar 只做上下文、主题、语言、工具入口，不重复路线导航
- command palette 只做跳转和搜索，不变成第二套导航
- 每个 route 只允许一个清晰的本地切换机制
- 移动端可以折叠 shell，但不能改变功能层级

## 3. 页面结构

每个工作台页面优先采用以下顺序：

1. 当前对象或领域身份
2. 一个最重要的下一步动作
3. mode/filter/task row
4. 主工作区
5. 折叠或浮层里的诊断、日志、历史和高级信息

避免：

- 大量同级 card 堆叠
- 首屏暴露过多实现细节
- route header、top tab、side task rail、object selector 同时抢导航职责
- 把日志、安装输出、健康检查结果做成永久卡片墙

## 4. 组件规则

### Button

- 使用有限变体：primary、secondary、danger、success、outline、ghost
- loading、disabled、focus、pressed 状态必须明确
- 主操作通常只有一个

### Form / Select

- label 在字段上方
- helper/error 在字段下方
- 字段背景必须是实色
- border、focus ring、disabled、invalid 状态必须可见
- dropdown 必须是 opaque sheet，不使用低对比透明面板

### Card / Row / Matrix

- card 只用于重复 browsable 记录
- 运维流程优先用 row、table、matrix、split inspector、drawer、runtime console
- 诊断事实优先放在数据矩阵，不做等高指标卡片墙

### Sheet / Dialog / Drawer

- 编辑、创建、日志、健康输出、命令结果优先使用 opaque sheet
- 页面可以 dim，但 sheet 本身必须清晰
- header/body/footer 分区要明确

## 5. 领域布局约束

### Agents

- 使用 roster selector + top task bar + active task canvas
- 本地任务包括 Overview、Persona、Routing、Sessions、Runtime
- Advanced JSON 保持折叠，不抢一线任务位

### Channels

- 使用 provider/account selector + top task bar + active task canvas
- provider 与 account 使用同一 selection model
- 用户文案说 routing/rules，不说 binding

### Studio Gateway

- 是模型网关和客户端接入管理面，不再使用 Codex Stack 命名
- Runtime / Gateway key / Active routing 保持高优先级
- Provider Center、App Connections、Smoke 使用 tabs 分区，低频运维动作进入更多菜单或折叠区
- 不恢复 CPA / Compact 旧诊断矩阵和安装修复复杂页面

### Chat

- `/chat` 是正式主聊天壳
- session list 固定在 shell 左侧或移动端 drawer
- inspector、record browser、slash status、debug 信息不得挤压主 conversation
- 富消息、附件、工具事件必须遵守 Studio contract，不展示 raw Gateway frame

## 6. CSS 所有权

- 全局 token 和 shared primitive 放在 `apps/web-vue/src/style.css` 与 `shared/styles/studio-workbench.css`
- feature CSS 只拥有本领域布局和本地 surface
- 不为同一 primitive 写多处 cascade patch
- 新 CSS class 应描述意图，例如 `rail`、`stage`、`matrix`、`sheet`、`runtime-console`
- 除非真的是重复内容卡片，不要继续扩散 `card` 命名

## 7. 验收检查

前端改动完成前至少检查：

- light/dark 都可读
- mobile 没有文本和控件重叠
- 表单字段有 label、helper/error、focus、disabled、invalid
- 页面没有重复导航
- 高风险动作有明确确认、失败态和恢复路径
- 日志和输出使用 sheet/runtime console，而不是永久卡片堆
