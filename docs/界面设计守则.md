# OpenClaw Studio 界面设计守则

> 状态：Active
> 更新：2026-06-12

## 1. 产品气质

Studio 是本地运维和 Agent 工作台，不是营销页。界面应安静、密集、清晰、可扫描，优先服务重复操作和状态判断。

## 2. 页面结构

- 按领域拆页：Gateway、Channel Connectors、Chat、Agents、System 等不要混成一个大页。
- 高频操作在主区，低频/危险操作放进更多菜单、折叠区或确认弹窗。
- 不使用大面积 hero、装饰渐变、浮动卡片堆叠。
- 页面 section 用 full-width band 或无框布局；卡片只用于重复 item、工具面板或 modal。

## 3. 组件规则

- 工具按钮优先图标 + tooltip。
- 模式选择用 segmented control / tabs。
- 二值设置用 toggle/checkbox。
- 数值用 slider/stepper/input。
- 列表项尺寸稳定，不因 hover、状态文本或动态图标抖动。
- 文本必须在移动和桌面视口内可读，不允许溢出遮挡。

## 4. 领域约束

### Studio Gateway

- Runtime/Gateway key/Active routing 是核心状态。
- Provider Center、App Connections、Smoke 分 tab 或清晰分区。
- 不内置具体 vendor preset；用户自己配置 baseUrl/key/model。
- 不恢复旧 CPA / Compact 诊断矩阵、安装修复复杂页面或路由模型残留文案。

### Channel Connectors

- 独立页面，不和 Gateway 堆在同一界面。
- 私聊配置、Agent Profile、平台 binding、runtime/session/log 分区清楚。
- Feishu/Octo 平台配置要贴近连接测试和运行状态。
- 群聊/管理类能力不作为当前主线，不在 UI 上制造复杂入口。

### Chat

- 以 IM 工作台为基准：会话栏、消息区、输入区、辅助 inspector。
- 消息宽度、代码块、表格和附件预览必须稳定。
- 运行中、排队、上传失败、断线恢复和停止状态要可见但不喧宾夺主。

## 5. 验收检查

- 桌面和移动视口无重叠、无横向页面溢出。
- 长文本、长路径、长 token、宽表格不撑破容器。
- 按钮组不会堆成一排不可读低频操作。
- 页面没有旧 Codex Stack / CPA / Compact 用户可见残留。
- 真实浏览器 smoke 或截图检查覆盖主要交互面。
