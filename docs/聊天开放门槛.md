# OpenClaw Studio Chat Open Gate

> 状态：SA-09C Updated
> 更新：2026-03-19

本文档回答：

什么时候 `OpenClaw Studio` 才算可以真正开放“私聊会话页”。

## 1. Technical Gate

### 要求

- `studio_managed` create / send / stream / abort / reset 稳定
- fresh live `agent_tool_call / agent_tool_result` 已打通
- `observed_external` 仍然只读
- `/api/chat/*` 与 `/ws/chat` same-origin 策略稳定
- 前端继续只消费 `Studio contract`
- 持久 session bridge 稳定

### 当前判断

- **大部分已满足**

已满足：

- text-first 主链路稳定
- fresh live tool stream 稳定
- readonly policy 稳定
- contract / transport / policy 稳定

未完全满足：

- `service.ts` 仍偏重，但这不阻塞开放面规划

## 2. UX Gate

### 要求

- `/chat` 不再是 workbench
- `/chat` 本身就是主聊天壳，而不是欢迎页
- 有固定 session list
- 新会话通过动作内 agent picker 创建
- 有标准聊天消息分组和 markdown/code render
- mobile 体验可用
- debug 信息不挤进主聊天页

### 当前判断

- **大体满足**

已满足：

- `/chat` 已变成统一 Chat Shell 主容器
- `/chat/s/:sessionRef` 已改成同壳 deep-link
- `/chat/workbench` 已改成同壳 inspect mode
- 正式聊天主面已去掉完整 debug/workbench 面板
- 左侧固定 session list 已回归
- 新会话时再选择 agent
- main conversation 已回到主舞台

仍待继续确认：

- 更长期的 mobile 细节打磨
- 更完整的开放前 UX 验收

## 3. Security Gate

### 要求

- `observed_external` 不应被当作开放聊天入口
- 正式私聊面默认只开放 `studio_managed`
- `deliver: false` 继续为默认
- 不把 Gateway raw frame 暴露给前端

### 当前判断

- **已满足**

## 4. Acceptance Gate

### 要求

- unit + integration tests 持续通过
- browser/API smoke 有正式命令入口
- 至少覆盖：
  - create / send / final
  - abort
  - reset
  - readonly policy
  - fresh live tool stream
  - reconnect

### 当前判断

- **已基本满足**

已满足：

- `npm run test:chat`
- `npm run smoke:chat:browser`
- `npm run smoke:chat:fresh-tool`

## 5. 当前缺口总结

当前主要剩余工作不再是底层基础设施，而是：

- 开放前的 UX polish
- route/sessionRef Phase 2 演进
- 更完整的开放验收

## 6. 当前是否建议开放

### 结论

- **现在可以开始面向更真实的私聊开放准备**

但还不建议直接把它视为“完全收口后的最终开放版”。

## 7. 什么时候建议开放

当以下条件满足时建议开放：

### 必须先完成

- `/chat` 已是正式私聊入口
- `/chat/workbench` 已收进 inspect mode
- deep-link route 已成型
- 主聊天页 Must-have now 已有第一版实现
- 再补一轮开放前 UX/产品验收

### 可以后续补

- 更丰富 tool cards
- usage 细化
- timeline 加强
- slash commands 提升

## 8. 当前结论

`chat` 基座已经够稳，且“开放页产品面”第一版已经完成。

因此当前阶段最合理的判断是：

- **技术基座已 ready**
- **开放面已进入可开放准备阶段，但还不是最终 fully-open 状态**
