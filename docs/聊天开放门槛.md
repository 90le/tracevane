# Tracevane Chat Open Gate

> 状态：Active Gate
> 更新：2026-06-12

## 1. Technical Gate

开放前必须满足：

- Chat typed API 可创建、读取、发送、停止、重置会话。
- WebSocket/SSE 断线后可短窗口补偿。
- Markdown、代码块、文件引用和媒体渲染不撑破布局。
- 长历史、队列发送、附件上传和草稿恢复有 smoke 覆盖。
- Gateway/OpenClaw 外部故障时，前端显示明确错误或 skip，而不是假成功。

当前判断：核心链路已可用，继续作为运行态迭代。

## 2. UX Gate

开放前必须满足：

- 会话 rail、消息区、composer、inspector 在桌面和移动端不互相遮挡。
- 长文本、长路径、长 token 和宽表格不会撑宽页面。
- 发送中、排队中、上传失败、运行失败和断线恢复状态清楚。
- 键盘操作和焦点恢复基本可用。

当前判断：单人模式已进入持续优化阶段。

## 3. Security Gate

开放前必须满足：

- 浏览器不直连 Gateway raw protocol。
- secret/token/upstream key 不进入前端或日志。
- 外部会话默认只读。
- 文件上传和资源 resolve 有路径、大小和权限边界。

当前判断：边界已明确，后续功能不能绕开 BFF。

## 4. Acceptance Gate

最小开放口径：

- Tracevane-managed chat 能完成真实对话。
- history / stream / abort / reset 行为一致。
- 附件和富消息能在常见浏览器显示。
- 断线恢复不会丢 final 状态或造成重复消息。
- 失败时可诊断。

## 5. 当前结论

Chat 可以作为 Tracevane 工作台功能继续迭代，但不应扩大到多人 room、生产频道默认写入或外部会话写入，直到单独完成授权、审计和持久事件模型。
