> 状态：Superseded baseline reference

本文保留为 Tracevane 全局产品背景，不再作为 Workspace 当前阶段的实现、设计或验收权威。Workspace 的当前阶段必须以 IDE-first 蓝图为准：先完成真实 AI 编程 IDE 工作区的主体布局、文件/编辑器、终端、Git、搜索、命令、证据和 Agent handoff，再评估写作、渲染、预览等未来扩展线。

## 1. 背景目标

Tracevane 长期方向仍是 **本地优先的 AI Agent Workspace 与连接控制层**。

它不是通用 OpenClaw 后台，不是普通文件管理器，也不是把终端、聊天、Provider 表单堆在一起的 admin console。长期产品应围绕一个项目工作区形成：

```text
选择上下文 → 编辑/运行 → 收集证据 → 审查 diff/命令 → 调用 Agent → 验证 → 回滚或交付
```

## 2. 当前产品支柱

| 支柱 | 当前边界 | 归属 |
| --- | --- | --- |
| Workspace | IDE-first 本地项目工作区：文件、代码编辑器、终端、Git、搜索、命令、证据、Agent handoff。 | 核心主入口 |
| Model Gateway | Provider、模型、协议、账号池、客户端接入、用量与路由证据。 | 连接层 |
| IM Channels | IM 平台账号、bot 凭据、绑定路由、投递会话、守护诊断。 | 连接层 |
| CLI Agents | Codex / Claude Code / OpenCode 等 CLI Agent runtime readiness、配置状态、Agent Run 生命周期和结果证据。 | Agent 运行层 |
| Platform | 第三方平台目录与平台原生管理，OpenClaw 是第一个平台。 | 系统/平台层 |
| Agent 会话 | 统一承载 Web / IM / CLI / OpenClaw / 未来平台触发的 Agent 对话、工具事件、审批和证据，可 handoff 到 Workspace。 | 工作层 |

## 3. Workspace 当前阶段边界

### 3.1 CLI Agent 不是终端

| 概念 | 负责什么 | 不负责什么 |
| --- | --- | --- |
| Terminal | shell session、cwd、命令输入输出、task runner、dev server、ports、terminal evidence。 | Codex/Claude/OpenCode runtime 管理。 |
| CLI Agents | CLI Agent 安装/版本/配置/readiness、Gateway 连接状态、Agent Run 生命周期、Agent 结果证据。 | 普通 PTY、shell tab、终端输入、终端 split/delete/rename。 |

Workspace 可以把 terminal 输出作为 Agent 上下文或证据，但普通 terminal session **不等于** CLI Agent Run。

### 3.2 文件管理属于 Workspace

新版本不再让 `/files` 与 `/workspace` 平级抢职责。

- 高频文件管理：Workspace Files / Explorer / File Manager。
- 审计型文件材料：Evidence / Artifacts。
- 如果保留 `/files`，它只能是证据/Artifacts 或兼容入口，不能再以“文件管理器”名义只读。

### 3.3 Workspace 是 AI 编程 IDE 工作面

当前阶段不追求复刻完整桌面 IDE，也不追求写作/预览/渲染体验。它追求比普通 IDE 更懂 Agent 工作流：

- 文件和 diff 是 Agent 修改的审查对象。
- 终端输出、日志、测试结果和 Git diff 是 Agent 上下文和验证证据。
- 高风险写入必须可审查、可回滚、可追踪。
- Command palette 只暴露真实 IDE 动作；不可用动作要禁用并说明原因。

## 4. 当前用户循环

### 4.1 本地项目修改循环

1. 打开 workspace/root。
2. 浏览或搜索文件。
3. 编辑文件或查看 diff。
4. 运行 task/terminal。
5. 查看 Git/Search/Terminal 证据。
6. 保存或交接证据。

### 4.2 Agent 协作循环

1. 选择文件、代码片段、终端输出、Git diff、测试结果或 IM 消息作为上下文。
2. 进入统一 Agent 会话，选择 Agent / 模型 / 工作目录 / 权限。
3. CLI Agents 记录 Agent Run 状态；模型网关提供模型路由；IM/平台只作为来源标签和投递证据。
4. Workspace 审查 Agent plan、patch、命令和验证结果。
5. 用户 approve/reject/apply/revert。

## 5. 响应式目标

Tracevane 必须同时是 PC 专业工作台和手机任务控制器。

| 设备 | 形态 | 目标 |
| --- | --- | --- |
| PC / Wide | 多栏 IDE：Explorer + Editor + Inspector + Bottom Panel。 | 高密度、可长时间工作、可 resize、快捷键/右键完整。 |
| Tablet | 双栏/抽屉混合：主舞台 + 可抽屉侧栏。 | 保持操作完整，减少同时可见区域。 |
| Mobile | 单栏任务流：Files / Editor / Terminal / Git / Search / Evidence 模式切换。 | 无横向溢出，所有 hover/右键能力有触摸替代。 |

## 6. 当前不作为 Workspace 主线

- 不建设独立写作产品线。
- 不做渲染增强、预览主题、富媒体阅读体验。
- 不把 Markdown 实时预览或所见即所得当作 Phase 0/1/2 验收目标。
- 不把“当前原型”作为实现门禁。
- 不新增说明页式、卡片墙式、第一季展示页式 Workspace。

## 7. 成功标准

1. 用户 3 秒内知道当前页面主对象是什么。
2. PC 无横向溢出；移动端无被挤压多栏。
3. 文件操作真实可用，不靠“去另一个页面”的说明文字。
4. 终端 session 可恢复、可重连、可清理 stale、可移动端使用。
5. Agent 改动必须形成 plan/diff/命令/验证/证据链。
6. CLI Agents 不管理普通 terminal；Terminal 不伪装 Agent Run。
7. 文档、导航、后端 API 和前端页面使用同一套边界语言。
8. OpenClaw 不拥有独立对话中心；所有人机对话统一进入 Agent 会话。
