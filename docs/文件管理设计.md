# 文件管理设计

> 状态：Active design
> 更新：2026-06-25

## 1. 定位

文件管理是 Workspace 的核心能力，不再是一个只读 `/files` 页面。

两种形态：

| 形态 | 用途 |
| --- | --- |
| Explorer Mode | IDE 左侧高频浏览、打开、上下文选择。 |
| File Manager Mode | 完整文件管理：表格/网格、批量操作、路径输入、上传下载、归档。 |

## 2. 必须能力

- 多 root 切换。
- 可编辑路径栏。
- 任意允许 root 内路径跳转。
- 树视图。
- 表格视图。
- 搜索文件名/内容。
- 新建文件/目录。
- 重命名。
- 复制/移动。
- 删除。
- 上传/下载。
- 批量选择。
- archive/unarchive。
- 大目录分页/虚拟滚动。
- 状态栏：数量、选中、路径、分页、Git 状态。
- 移动端 action sheet。

## 3. UI 规则

PC：

```text
toolbar + path bar
left/tree or table
right optional inspector
status bar
```

Mobile：

```text
top path/search
single list/table cards
selection mode
bottom action sheet
```

不要再出现“本页只读，去 IDE 操作”的说明。

## 4. 与 Evidence 的关系

文件管理负责操作文件；Evidence 负责保存和复盘文件相关证据。

示例：

- 下载文件：文件管理动作。
- 查看某次 Agent 生成的 artifact：Evidence。
- 对 Git diff 做审查：Workspace Git/Editor。
- 查看 IM 附件：Evidence，可 reveal/open in Workspace。

## 5. 安全和确认

| 动作 | 模式 |
| --- | --- |
| 新建/重命名 | inline/dialog，失败 toast。 |
| 覆盖/移动/删除 | confirm dialog/sheet。 |
| 批量删除 | 列出 affected paths。 |
| archive/unarchive | 显示目标目录与文件数量。 |
| upload | sheet + progress + overwrite policy。 |
