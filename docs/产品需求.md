# OpenClaw Studio PRD

> 状态：Gateway + Channel Connectors 主线 active
> 更新：2026-06-12

## 1. 产品目标

OpenClaw Studio 是 OpenClaw 的本地管理控制台，用来替代手写配置、散落 CLI 和记忆性运维操作。

当前阶段的主线是：

- 保持管理域稳定可用。
- 用 Studio Gateway 提供统一模型中转、provider 管理和 App Connections。
- 用 Channel Connectors 提供 Feishu/Octo 等 IM 私聊到 CLI Agent Bot 的原生链路。
- 保持 Chat 作为 Studio typed 工作台，不直接暴露 Gateway raw protocol。

## 3. 目标用户

- 单机自托管 OpenClaw 的个人用户。
- 管理多 Agent、多频道、多技能、多 cron 任务的重度用户。
- 希望 Codex、Claude Code、OpenCode、OpenClaw 等 CLI 工具统一走本地模型网关的开发者。
- 希望通过 Feishu/Octo 私聊使用本地 CLI Agent 的用户。

## 4. 产品原则

- 配置界面化，不鼓励直接改 JSON。
- 页面按领域拆分，不把多个域重新揉进一个大页。
- 共享类型先行，前后端语义保持一致。
- 写操作保守，优先合法、可预览、可回滚。
- 高风险能力必须有确认路径和清晰错误态。
- 浏览器不直接消费 Gateway raw protocol；前端只消费 Studio typed contract。
- Channel Connectors 新能力先迁移 CC Go / OpenClaw 成熟 contract，再做 Studio 精修。

## 5. 页面范围

| 页面 | 当前职责 |
| --- | --- |
| Dashboard | 汇总关键状态和入口 |
| Config | 系统级配置、provider secret、默认模型等 |
| Skills | 技能市场、安装、更新、启停、配置 |
| Terminal | PTY/WebSocket 维护终端、CLI 工具链状态 |
| Channels | OpenClaw 频道/账号/binding 管理 |
| Cron | 定时任务 CRUD、运行记录、失败通知 |
| Agents | Agent 列表、文档、binding、session、ACP 基础配置 |
| System | 健康、诊断、版本、事件、升级与修复入口 |
| Chat | Studio 自管 chat 工作区 |
| Studio Gateway | Gateway daemon、Provider Center、App Connections、模型/路由/smoke |
| Channel Connectors | IM 私聊平台、Agent Profile、binding、runtime、session/log |

## 6. 当前成功标准

- 管理域常见读写操作不依赖手写配置。
- Studio Gateway 能让三类 provider 原生协议对外转换为三类常见客户端协议。
- App Connections 能生成 Codex、Claude Code、OpenCode、OpenClaw 的脱敏 preview/apply/rollback。
- Channel Connectors 私聊可稳定触发本地 CLI Agent，并通过 Gateway 使用模型。
- Feishu/Octo 私聊支持文本、文件、图片、工具流、过程回复、最终回复和核心控制命令。
- Studio/OpenClaw 崩溃时，Gateway daemon 与 Channel daemon 仍可继续服务。
- 文档与当前边界一致，不再追加旧 Codex Stack / CPA / Compact 或 platform action 方案。

## 7. 当前不做

- 旧 Codex Stack / CPA / Compact 用户链路。
- 托管 `cc-connect` 作为生产路径。
- Feishu/Octo 平台 API 工具层和 `studio-channel-skill`。
- 群聊、多 bot 协作、thread 管理的首期产品化。
- 新建 Clawith 式平行数据库会话模型。
- 浏览器直连 Gateway raw frame。
