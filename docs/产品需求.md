# Tracevane Product Requirements

> Status: active baseline
> Updated: 2026-06-25

## 1. Product goal

Tracevane is a local-first AI Agent Workspace and connectivity control layer.

It should let a user safely answer and operate:

- Which local workspace/project am I working in?
- Which files, diffs, terminal outputs, previews and evidence belong to this task?
- What did an Agent propose, change, run, verify or fail to do?
- Which model/provider/account is each Agent or client using?
- Which IM account/bot/binding receives messages and where are replies delivered?
- Which daemon, local config or platform substrate needs repair?

## 2. Current product judgment

| Area | Current state | Product judgment |
| --- | --- | --- |
| Workspace / IDE / Files / Terminal | Rebuilt as a new shell but currently incomplete and split across `/ide` and `/files`. | Must be redesigned as one Workspace system, not patched page by page. |
| File Management | Backend mutation APIs exist, but `/files` is read-only evidence while IDE has hidden context actions. | Move true file management into Workspace; keep evidence/artifacts separate. |
| Terminal | xterm/node-pty path exists, but session recovery/stale handling/mobile input are insufficient. | Upgrade to Terminal Runtime System inside Workspace. |
| CLI Agents | Runtime projection exists and must stay separate from terminal. | Own Agent runtime/readiness/runs only; do not manage terminal. |
| Model Gateway | High completion. | Stabilize and integrate with Agent/Workspace handoff via links, not by taking over Workspace. |
| IM Channels | Needs full account/binding/test workflow. | Continue as IM owner domain. |
| Platform | OpenClaw platform workspace exists. | Keep platform-native management separate from Workspace and Gateway/IM owners. |

## 3. Core domains

### Workspace

Owns the local project work surface:

- Files explorer and full file manager.
- Code/Markdown/config/diff editors.
- Preview for Markdown, static HTML and local web apps.
- Terminal runtime, tasks, logs and ports.
- Git changes, hunk/diff review and commit/revert flows.
- Evidence basket: screenshots, logs, diffs, artifacts and verification output.
- Agent handoff: selected context, proposed changes, command review and verification.

### Terminal

Terminal is a Workspace tool, not CLI Agents.

Owns:

- shell/PTY sessions;
- input/output;
- session recovery/replay;
- task runner;
- dev server/ports/logs;
- terminal evidence.

### CLI Agents

Owns Codex / Claude Code / OpenCode runtime and Agent Run lifecycle:

- install/version/config readiness;
- Gateway connection readiness;
- launch/resume/stop boundary for Agent tasks;
- Agent Runs from Workspace, Chat or IM sources;
- Agent result evidence links.

Does not own ordinary terminal tabs or PTY management.

### Model Gateway

Owns provider/model connectivity, protocol adaptation, routing, account pool, app connection config, usage/cache evidence and gateway daemon.

### IM Channels

Owns IM platform account, bot credentials, binding routes, inbound handling, attachments, outbound delivery and channel daemon.

### Platform

Owns third-party platform management. OpenClaw is first platform: guard, config, agents, skills, channels, bindings, services, logs and diagnostics.

## 4. Non-goals

- Do not build a generic OpenClaw admin clone.
- Do not present half-built pages with user-visible placeholders.
- Do not split file management into contradictory `/ide` write and `/files` read-only surfaces.
- Do not mix Terminal with CLI Agents.
- Do not expose secrets to preview iframes or Agent context by default.
- Do not allow Agent file writes without diff/approval in high-risk paths.

## 5. Near-term priorities

### P0

1. Create the Workspace-first shell and responsive layout: PC multi-pane, mobile single-task flow.
2. Move true file management into Workspace: path bar, batch ops, upload/download, copy/move/delete/archive.
3. Upgrade Terminal Runtime System: replay, reconnect, stale cleanup, mobile input toolbar.
4. Update CLI Agents so terminal wording/control is removed and Agent Run ownership is clear.

### P1

- Diff editor and Git hunk review.
- Preview console/screenshot evidence.
- Evidence basket.
- Workspace → CLI Agents handoff records.

### P2

- Agent plan/patch/command approval.
- Verification checklist.
- Workspace task history and evidence bundle export.
