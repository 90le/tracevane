# 终端能力设计

> 状态：Active design
> 更新：2026-06-25

## 1. 定位

终端是 Workspace 的本地运行工具，不是 CLI Agent。

它负责 shell、命令、task、dev server、logs、ports 和 terminal evidence。

它不负责 Codex/Claude/OpenCode runtime readiness、Agent Run 生命周期或模型路由配置。

## 2. 目标

终端能力必须比当前版本更稳：

- session 可恢复；
- 输出可 replay；
- 断线可重连；
- stale session 自动清理；
- 移动端可实际输入；
- task/ports/logs 一等化；
- 输出可加入证据或 Agent 上下文。

## 3. 前端能力

PC：

- session tabs；
- rename；
- new/duplicate；
- close/kill；
- reconnect；
- clear local view；
- search output；
- copy selected output；
- fullscreen；
- send output to evidence/context；
- task launcher；
- ports inspector。

Mobile：

- full-screen terminal；
- input accessory bar；
- paste command；
- Ctrl+C；
- Esc/Ctrl/Tab/Arrows；
- reconnect/clear/copy/search in action sheet；
- large touch targets。

## 4. 后端能力

必须有：

- registry；
- pty lifecycle；
- output ledger；
- SSE/WebSocket reconnect；
- replay from seq；
- heartbeat；
- stale cleanup；
- bounded buffer；
- resize debounce；
- structured errors。

## 5. 错误体验

禁止首屏直接显示裸错误如：

```text
[error: terminal_session_not_found]
```

应该显示：

- 会话已失效；
- 可“新建终端”或“从列表移除”；
- 若可恢复，提供“重连”；
- 错误详情折叠到 evidence/log。

## 6. 与 Agent 的关系

允许：

- terminal output 作为 Agent context；
- terminal command result 作为 verification evidence；
- Agent 建议命令可在 Terminal 中经确认执行。

禁止：

- terminal session = CLI Agent Run；
- CLI Agents 管理 terminal tabs；
- terminal 页面显示 Provider/模型配置。
