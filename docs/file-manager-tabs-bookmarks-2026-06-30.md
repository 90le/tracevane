## 2026-06-30 多级收藏夹补充

- 用户反馈：收藏夹应像浏览器书签管理一样支持多级文件夹，而不是把收藏路径平铺成标签。
- 本次决策：保留本地持久化，但把收藏数据从扁平路径数组升级为 `folder/bookmark` 树；旧扁平数组在读取时自动迁移成根级收藏，避免用户已有收藏丢失。
- 交互边界：收藏夹支持根级/子级新建文件夹、收藏当前路径到任意文件夹、打开、重命名、删除；删除文件夹会删除其子收藏，符合浏览器书签文件夹的管理心智。
- 性能边界：收藏夹树限制为 200 个节点，读取时规范化并裁剪；文件浏览、索引和回收站仍不依赖收藏夹存储成功。
- 拒绝方案：继续使用扁平路径列表并用路径前缀模拟分组；该方案无法表达用户自定义书签文件夹和命名。
- 验证计划：运行 Web TypeScript 检查和 diff whitespace 检查，重点保护旧收藏迁移、导航标签、收藏树递归渲染的类型契约。

## 2026-06-30 完整收藏夹管理器补充

- 用户反馈：收藏夹不是机械式按钮列表，应接近浏览器书签管理能力，支持树形菜单折叠、拖拽移动、排序和完整管理弹层。
- 本次决策：将收藏夹入口升级为内部管理器面板，文件夹可展开/收起；书签和文件夹行支持拖拽，拖到行上方/下方调整顺序，拖到文件夹中移动层级。
- 交互边界：新建文件夹、收藏当前、重命名均使用 Tracevane 内部弹层，不再调用浏览器 `prompt`，避免原生弹窗阻塞和破坏整体产品体验。
- 数据边界：仍使用同一收藏树持久化结构，移动操作通过先抽取节点再插入目标父级/顺序实现，并禁止把父文件夹移动到自身子级。
- 验证计划：运行 Web/API TypeScript 检查与 diff whitespace 检查；拖拽交互为浏览器原生 HTML5 DnD，无新增依赖。
