# Tracevane Chat Contract

> 状态：Active
> 更新：2026-06-26

## 1. 原则

- 前端不直接消费 Gateway raw frame。
- 前端只消费 Tracevane typed contract。
- 后端负责把 Gateway / OpenClaw chat、agent、tool 事件归一化。
- MVP contract text-first，但保留富消息、附件和 observability 扩展点。

## 2. Session Kind

```ts
type ChatSessionKind =
  | "tracevane_managed"
  | "observed_external"
  | "system_internal";
```

含义：

- `tracevane_managed`：Tracevane 创建并可写的会话。
- `observed_external`：观察外部 Gateway/OpenClaw 会话，默认只读。
- `system_internal`：cron、heartbeat、系统 run 等内部会话，不进普通 rail。

## 3. 核心资源

```ts
interface ChatSessionRow {
  key: string;
  agentId: string;
  sessionId: string | null;
  kind: ChatSessionKind;
  label: string;
  derivedTitle: string | null;
  lastMessagePreview: string | null;
  updatedAt: string | null;
  permissions: {
    writable: boolean;
    canSend: boolean;
    canAbort: boolean;
    canReset: boolean;
    canDelete: boolean;
    visibleInFrontend: boolean;
  };
  runtime: ChatRuntimeState;
}

type ChatMessageRole = "user" | "assistant" | "system" | "tool" | "unknown";

interface ChatMessageItem {
  id: string;
  role: ChatMessageRole;
  text: string;
  createdAt: string | null;
  source: "history" | "stream" | "inject";
  runId: string | null;
  truncated: boolean;
  omitted: boolean;
  aborted: boolean;
  stopReason: string | null;
}
```


## 3.1 Runtime Target

每个 `ChatSessionRow` 带有 `runtimeTarget`，用于表达该会话未来应该由哪个运行器处理：

```ts
interface ChatSessionRuntimeTarget {
  adapterKind: "openclaw-gateway" | "native-cli";
  agent: string;
  model: string | null;
  workDir: string | null;
  permissionMode: string | null;
}
```

`runtimeTarget` 是 typed metadata，不是直接执行凭据；前端可以展示/配置它，后端 adapter 负责把它翻译成具体 CLI 或平台调用。

## 4. API Contract

| API | 作用 |
| --- | --- |
| `GET /api/chat/health` | Chat 可用性和 runtime 摘要 |
| `GET /api/chat/bootstrap` | 首屏会话、配置和诊断 |
| `GET /api/chat/agents/:agentId/sessions` | Agent 会话列表 |
| `POST /api/chat/agents/:agentId/sessions` | 创建 Tracevane-managed 会话 |
| `GET /api/chat/sessions/:sessionKey/history` | 分页历史 |
| `POST /api/chat/sessions/:sessionKey/send` | 发送消息，可携带 `fileRefs` |
| Files API `/api/files/summary`、`/api/files/browse`、`/api/files/read`、`/api/files/uploads/*`、`/api/files/download` | 目录选择、文件上传、文件预览/下载；Chat 只消费 `files:<rootId>:<path>` 引用 |
| `POST /api/chat/sessions/:sessionKey/abort` | 中止当前 run |
| `POST /api/chat/sessions/:sessionKey/reset` | 重置会话 |
| `WS /ws/chat` | 实时事件、短窗口 replay、runtime update |

明确不属于 Chat API：

| API/能力 | 去向 |
| --- | --- |
| `/api/chat/sessions/:sessionKey/controls` | 删除；权限/审批归具体 runtime 或平台安全域 |
| `/api/chat/sessions/:sessionKey/slash-gateway` | 删除；OpenClaw 原生管理代理迁至 `/api/platforms/openclaw/gateway` |
| Gateway raw methods (`models.list`, `exec.approvals.*`, `sessions.patch`, `sessions.compact`, `tools.effective`) | Platform/OpenClaw API 或未来 runtime adapter，不由 Chat session key 隐式代理 |


## 4.1 Composer 文件附件

- Chat 不再拥有独立上传 API，也不再调用旧的 `/api/chat/sessions/:sessionKey/upload`。
- 目录加载、文件选择、上传、预览和下载统一走 Files API：`summary` 取根目录，`browse/read` 做选择/预览，`uploads/init -> chunks -> complete` 做二进制分片上传，`download` 生成 inline/attachment 链接。
- Composer 本地上传默认写入 `project-root/.tracevane/chat-uploads/<session>/...`；如果当前环境没有 `project-root`，才回退到 Files API 的 `defaultRootId` / preferred root。这样 Chat、Workspace、File Manager 的新文件 API 切换后仍优先落在当前工作区，而不是误写入 OpenClaw 平台根。
- Composer 发送消息时只通过 `ChatSendRequest.fileRefs` 传 `files:<rootId>:<path>`，并保留 `rootId` 字段，避免新 File Manager/Workspace 多根 API 切换后把非项目 root 附件误当成 `workspace:`；后端按 Files API 安全解析，并把结构化 `resources` 写回历史。
- 前端渲染 `resources` 时只消费 typed `ChatResourceItem`，不解析 OpenClaw/Gateway 原始附件字段，不回退到 `uploads:` 私有引用。


## 4.2 Runtime Adapter 内部合同

`POST /send` 的后端实现不直接绑定 OpenClaw Gateway。ChatService 先调用内部 `ChatRuntimeAdapter.send()` / `ChatRuntimeAdapter.abort()` / `ChatRuntimeAdapter.reset()`：

- 当前默认 adapter：`native-cli`，用于 Codex / Claude Code / OpenCode 等本地 agent session driver。
- 兼容 adapter：`openclaw-gateway`，只为兼容现有 OpenClaw 会话桥和历史 WebChat transcript，封装 `chat.send`。
- `send` 返回值只允许映射成 `ChatSendStatus` 与 `runId`；`abort` 返回值只允许映射成 `aborted` 与 `runIds`；`reset` 返回值只允许映射成 reset ok 状态；`deleteSession` 返回值只允许映射成 delete ok 状态；`listSessions` 与 `readHistory` 返回值只允许映射成 rows/messages；前端不读取 adapter kind 或 raw CLI/Gateway 字段。

这个边界是为了防止旧 OpenClaw WebChat 能力再次混入统一 Agent 会话层。新增 runtime 能力必须先归一成 Chat typed event / typed API，再暴露给前端。

## 5. Stream Event

前端只接收归一化事件：

- `message.delta`
- `message.final`
- `tool.started`
- `tool.delta`
- `tool.completed`
- `tool.failed`
- `runtime.updated`
- `run.started`
- `run.completed`
- `run.failed`
- `run.aborted`
- `replay.completed`

不允许前端直接依赖：

- provider raw chunk
- Gateway raw frame
- OpenAI/Anthropic/Codex/OpenClaw 私有字段
- 未归一化 tool payload

## 6. 错误语义

错误必须映射为稳定 shape：

```ts
interface ChatContractError {
  type: string;
  code: string;
  message: string;
  retryable: boolean;
  details?: unknown;
}
```

原则：

- 对用户显示可读 message。
- 对调试保留脱敏 details。
- 不泄露 secret、token、upstream key。

## 7. 冻结结论

Chat contract 继续服务 Tracevane 自管会话和短窗口实时恢复。多人/群聊、持久事件总线和跨端协作不在当前阶段。
