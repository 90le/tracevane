# 参考原型审计：chat-workbench-rebuild-20260617

> 日期：2026-06-18
> 目标：吸收参考目录所有页面的优点，明确丢弃缺陷，为 React + shadcn/ui + Tailwind CSS 新前端原型收敛方向。

## 结论

新的 Tracevane 方向不照抄任何单页，而是组合这些稳定优点：

- `app-shell-v2.html` 的单栏分组导航：核心页面靠文字 + 图标分组可见，折叠后才使用 tooltip。
- `chat-workbench-v3.html` 的克制材质、左主对象列表、中央工作区、右侧检视器、命令入口和右键菜单。
- `agents-prototype.html` 的 master-detail：列表是主对象，选中后详情用 tabs 分维度。
- `model-gateway-prototype.html` 的列表优先 + 抽屉详情：敏感配置和低频配置不在主屏平铺。
- `terminal-ide-v2.html` 的密集区处理：代码、终端、文件树用实色和细边框，材质退后。
- `dashboard-prototype.html` 的主状态条 + 关键指标 + 服务状态，但需要并入全局工作台外壳。

## 必须丢弃的缺陷

- 不再使用纯 macOS 材质派作为唯一方向；上一轮已证明单一路线风险高。
- 不使用 VS Code 式全局活动栏作为主导航；核心页面只靠图标记忆成本太高。
- 不使用 emoji 作为正式导航/工具图标；未来 React 落地用 `lucide-react`。
- 不使用固定 `width=1440` 的 viewport；原型也必须响应桌面和移动。
- 不把毛玻璃用于表格、终端、日志、代码编辑器正文；信息密集区用实色。
- 不把说明文案放满首屏；原型阶段界面语言以中文为主，技术名词保留英文。
- 不做卡片墙；卡片只用于少量状态摘要或重复对象。
- 不把列表、详情、配置、日志全部平铺；配置/低频信息进 Sheet/Drawer/Dialog。

## 新综合方向

名称：**Tracevane shadcn 工作台**

一句话：以 shadcn/ui + Tailwind CSS 的现代 React 组件模型为底，保留单栏分组导航和对象检视器，视觉上使用舒适的中性底、克制圆角、柔和状态色和清晰行表布局。

## 2026-06-18 补充参考

本轮（历史，V8 已弃用）收敛到旧 V8 基线前，按用户反馈增加外部设计系统参考。重点不是复制视觉，而是吸收信息架构处理方式：

- shadcn/ui：`Sidebar` / `Sheet` / `Dialog` / `Data Table` / `Command` 的组合适合作为 React + Tailwind 落地基线。结论：对象详情和设置优先进入 `Sheet`，不要在仪表盘主页平铺详情卡。
- Shopify Polaris：Index table / resource list 类模式强调对象索引、批量操作、分页和进入对象详情。结论：主页主对象应是可扫描列表，而不是卡片墙。
- Atlassian Design System：Drawer / Modal / Page layout 的分层适合处理辅助信息和低频配置。结论：详情、诊断、确认动作不应抢占主页面。
- Microsoft Fluent 2：导航、工具条、表格和面板强调任务上下文与渐进披露。结论：全局导航和命令入口保留，配置入口改为设置 Sheet。

本轮落地调整：

- 删除主页侧边/下方详情卡；点击待处理对象或服务对象直接打开对象 Sheet。
- `配置入口` 改为 `设置` Sheet，作为低频层，不再像主操作一样挤在首屏说明中。
- 仪表盘主页只保留运行态势、关键指标、主对象索引。
- 服务和事件放入页签，作为同页面二级层；完整配置和日志继续下沉。
- 多配色在同一原型内通过设置 Sheet 预览，避免生成一堆结构重复的 HTML。

## 2026-06-18 模块化方案废弃记录

曾短暂尝试把首页按系统能力和数据来源重新组织。该方案虽然覆盖完整，但仍然让旧前端路由和后端模块决定用户看到什么，打开首页时更像系统说明而不是工作入口。该方向已被下面的“任务优先修正”取代，不再作为候选。

## 2026-06-18 任务优先修正

用户反馈“功能域、运行域必要性不大”，并明确不要被系统原有前端和后端限制原型设计。因此最新原型继续收敛：

- 删除“功能域地图”和“核心运行域”作为首页结构。
- 首页主语言改为：今日简报、下一步、快速启动、正在进行、近期动态。
- 技术模块只作为数据来源，不作为用户可见的信息架构。
- “模型网关”在首页保持为用户能直接理解的连接状态，而不是展示底层 API 模块。
- “渠道连接 / 外部链接”在首页表达为消息入口、会话调度、外部服务和接入路径。
- 后端建议从 `/api/dashboard/summary` 的模块摘要进一步演进成 `/api/home/briefing` 这种任务级聚合，而不是让前端直接拼内部模块目录。

## 2026-06-18 首页驾驶舱修正

用户进一步指出：当前仪表盘虽然不叫“功能域/运行域”，但整体布局和内容又回到了 Tracevane 现有版本的侧栏、模块和管理台主页结构。随后补充“左侧导航栏还是需要的”，因此新版做出两点区分：

- 保留左侧导航：采用 shadcn Sidebar 类结构，包含 workspace header、命令入口、分组导航、footer 工具和折叠态。
- 不让左侧导航决定首页：首页主舞台不按导航组、后端模块或旧页面路由排布，而是按工作简报、优先队列、快速启动、任务泳道、继续入口、短事件和底部信号组织。

本轮通过 shadcn MCP 显式查询了官方 registry 示例：

- `sidebar-demo` / `sidebar-06`：确认 Sidebar 可由 header、content、group、footer、rail、trigger 组合，不必退回旧 Tracevane 模块目录。
- `sheet-demo`：确认从属详情适合 `SheetHeader` / `SheetContent` / `SheetFooter` 分层，首页不常驻详情。
- `command-demo`：确认命令面板适合 `CommandInput`、`CommandGroup`、`CommandItem`、shortcut 分组，作为导航补充。

新版原型据此调整：

- 命令面板改成分组结构，而不是普通按钮列表。
- 对象详情 Sheet 改成 header / body / footer；操作固定在底部，不挤在正文 section 内。
- 危险或低频动作使用 Dialog，例如本地设备审批、恢复检查。
- 工作流使用泳道而不是表格主页，避免视觉上回到旧后台。

## 2026-06-18 舒展仪表盘修正

用户反馈上一版仍然太密集，并且仪表盘不应写成介绍页。本轮继续收敛：

- 删除 UI 中的解释性文案，例如“不再复刻旧版”“像大厂控制台”“只回答三个问题”等。
- 首屏只保留真实状态：工作区运行状态、待处理事项、关键指标。
- 快速启动从 6 个减少到 4 个，降低按钮墙感。
- 删除三泳道任务板和底部指标块，改为更舒展的“继续工作 + 运行信号 + 短事件”。
- 视图设置只显示主题、配色、密度和交互偏好，不再在产品 UI 中讲设计依据。

## 2026-06-18 接入态势修正

用户继续反馈：左侧导航还有不必要项；“继续工作 / 运行信号 / 短事件 / 待处理”存在重复；仪表盘应更关注 Tracevane 独立产品自身的数据，尤其模型网关、渠道链接、外部链接。

本轮调整后来继续演进为 V8 durable shell，最终保留的是应用壳气质而不是这些旧分区本身：

- 左侧导航删除工作区卡片、文件、技能、工作流、定时任务、插件、健康、记忆、设置等低频或重复项，改向总览 / 运行 / 连接 / 证据 / 系统五组收敛。
- 仪表盘主数据改为模型网关、渠道链接、外部链接，弱化 OpenClaw 本地工作区数据。
- 删除重复的接入概览对象卡，改为“接入路径 + 24 小时流量分布”，让中段表达链路关系而不是复述顶部指标。
- 合并“待处理”和“接入动态”为“关注队列”，默认显示待处理，动态进入同面板标签页。
- 顶部状态卡、快速启动、接入路径、流量分布均使用 `repeat(auto-fit, minmax(...))` 或等价动态网格，让布局随宽度自动铺满。

落地映射：

| 原型概念 | React/shadcn 落地组件 |
| --- | --- |
| 全局单栏导航 | `Sidebar` + `Collapsible` + `Tooltip` |
| 命令面板 | `CommandDialog` / `CommandGroup` / `CommandItem` |
| 页面顶栏 | `Button` / `Breadcrumb` / `DropdownMenu` |
| 状态摘要 | `Card` 但数量受控 |
| 关注队列 | `Tabs` + `Button` / `Card` / `Badge`，点击进入 `Sheet` 或 `Dialog` |
| 接入路径 | `Card` / `Button` / `Progress` 或等价分段面板，不是首页数据表 |
| 选中对象详情 | `SheetHeader` / `SheetContent` / `SheetFooter`，不在 Dashboard 首屏常驻 |
| 维度切换 | `Tabs` |
| 低频配置 | `Sheet` / `Dialog` / `Accordion` |
| 代码/终端/日志 | 专用实色 surface，不套玻璃卡片 |

## 验收标准

- 打开页面 3 秒内能判断主对象：模型网关、渠道链接、外部链接是否可用。
- 高频动作不超过 2 步：刷新、进入维护终端、处理待办、打开对象详情必须可见。
- 低频配置默认不可见：设置、Provider key、Agent 高级 JSON 等进入抽屉或详情页。
- 中文为主：设计原型界面和说明文档默认中文，只有库名、协议名、命令、路径保留英文。
- 深浅主题都清晰：文字、边框、hover、focus、状态色不过暗不过曝。
- 移动视口不出现横向页面滚动；导航变为抽屉或顶部紧凑入口。
