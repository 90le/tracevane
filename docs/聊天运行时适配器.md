# Tracevane Chat Runtime Adapter 设计记录

> 日期：2026-06-25
> 状态：Active

## 目标

Chat 现在是统一 Agent 会话层，不再是旧 OpenClaw WebChat 页。它需要同时承载：

- Codex / Claude Code / OpenCode 等本地 CLI Agent；
- OpenClaw 等三方平台 Agent；
- IM 渠道投递过来的会话；
- 后续其它平台或自定义 Agent。

因此 Chat 后端必须有一个“运行时适配器”边界：前端和 Chat typed API 只理解统一会话、消息、附件、运行状态；具体运行器差异在 adapter 内部处理。

## 研究记录

- OpenAI Codex CLI 官方帮助文档（2026-06-25 检索）：Codex CLI 是本地终端 coding agent，并通过 ChatGPT/Codex 登录使用；Tracevane 不应把它建模成 OpenClaw Gateway raw session。
- Claude Code 官方 Sessions/Permissions 文档（2026-06-25 检索）：Claude Code 会话、权限模式和工具执行权限属于 Claude Code runtime/项目环境；不应抽象成 Chat 通用 `host-exec controls` 页签。
- OpenCode 官方 CLI 文档（2026-06-25 检索）：OpenCode 支持 `--continue`、`--session`、`--model`、`--agent`、`run`、`agent` 等 CLI 级 session/agent 能力；Tracevane 应通过 runtime adapter 选择 session/model/agent，而不是走 OpenClaw slash gateway。
- 本地现状：`apps/api/modules/channel-connectors/*driver.ts` 已有 Codex/OpenCode/Claude 风格会话驱动线索；`apps/api/modules/chat/service.ts` 仍有 send/history/list/reset 等桥接调用，需要分阶段从 OpenClaw Gateway bridge 迁到 typed runtime boundary。

## 当前实现边界

当前已落地最小接口：

```ts
type ChatRuntimeAdapterKind = 'openclaw-gateway' | 'native-cli';

interface ChatRuntimeAdapter {
  kind: ChatRuntimeAdapterKind;
  send(input: ChatRuntimeSendInput): Promise<ChatRuntimeSendResult>;
}
```

已落地 `send` 与 `abort` 两条最小路径：

```text
ChatService.performDirectSend
  -> createCurrentChatRuntimeAdapter()
  -> adapter.send(...) / adapter.abort(...)
  -> normalizeChatRuntimeSendResult(...) / normalizeChatRuntimeAbortResult(...)
  -> ChatSendAck / ChatAbortResponse / runtime projection / stream ack
```

当前 adapter kind 是 `openclaw-gateway`，只包住现有 `chat.send` 与 `chat.abort`，保持行为不变。这样先建立扩展缝，再逐步接 Codex / Claude Code / OpenCode native driver。

## 必须删除/禁止回流的旧能力

以下能力不属于统一 Chat：

- `/api/chat/sessions/:key/slash-gateway`
- `/api/chat/sessions/:key/controls`
- `session.controls` stream event
- `host-management-exec` 作为 Chat 默认控制页签
- `models.list` / `skills.status` / `exec.approvals.*` 等 raw OpenClaw 管理命令

这些能力的归属：

- OpenClaw 原生管理：Platform / OpenClaw 子域；
- Agent 权限：具体 runtime / CLI driver / 平台安全域；
- 模型选择：模型网关或 runtime adapter 的 typed 配置；
- 文件：Chat upload/resource typed contract，不能泄露 provider raw attachment 字段。

## 后续任务

1. 把 history/list/reset 从“隐含 OpenClaw bridge”逐步收敛到 adapter 方法；abort 已先进入 adapter。
2. 给 Codex / Claude Code / OpenCode 建立 native adapter：选择工作目录、agent、模型、session resume/fork/continue。
3. 前端会话列表只显示统一语义：可管理会话、外部观察会话、文件夹、归档；不要显示 provider raw session key。
4. 运行详情 Drawer 只显示证据/队列/诊断，不恢复 OpenClaw 控制页签。
5. 每新增一个 runtime adapter，必须补充 focused smoke：send、abort、history replay、附件或文件上下文。
