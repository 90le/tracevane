# Tracevane Chat Markdown/HTML/SVG 混合渲染方案

> 状态：Active Reference
> 更新：2026-06-24

## 1. 当前结论

保留现有安全渲染原则：Markdown / HTML / SVG / Mermaid 必须经过受控 display adapter、sanitizer 和隔离预览，按需支持 Markdown 正文中的 HTML/SVG mixed render。

不为了此需求迁移到 `markdown-it`、`remark/rehype` 或新渲染栈。

## 2. 当前链路

- `apps/web/src/features/chat/ChatWorkbenchPage.tsx` 承载 Chat 工作台。
- `apps/web/src/features/chat/views/ConversationView.tsx` 渲染会话消息。
- 富消息/Markdown 渲染应继续走统一 sanitizer 与 preview 组件，不在普通正文 DOM 中直接信任 raw HTML/SVG。
- HTML/SVG/Mermaid fenced code block 继续走隔离 preview 增强。

## 3. 策略

- 普通 Markdown 默认渲染。
- fenced `html` / `svg` / `mermaid` 继续走隔离 preview。
- raw HTML/SVG mixed render 只在明确开关允许时启用。
- sanitizer 分层处理 Markdown、raw HTML fragment、raw SVG fragment。
- 复杂 HTML 继续建议用 fenced code preview，不直接塞进正文 DOM。

## 4. 不迁库原因

当前项目已有：

- Chat 工作台与会话视图。
- Tracevane 媒体 token / resource resolve。
- copy button、preview、富消息组件。
- DOMPurify / sanitizer 安全规则。
- 全局/会话级开关。

迁库收益有限，改造面更大。

## 5. 验收

- 宽表格、长代码块、inline HTML、SVG 不撑宽 chat pane。
- raw HTML/SVG 关闭时继续转义显示。
- raw HTML/SVG 开启时不允许脚本、事件 handler、危险 URL 和样式污染。
- fenced preview 能继续全屏、复制和隔离渲染。
