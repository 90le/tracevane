# Studio Chat Markdown/HTML/SVG 混合渲染方案

> 状态：Active Reference
> 更新：2026-06-12

## 1. 当前结论

保留现有 `marked + DOMPurify + MarkdownBlock.vue` 架构，按需增量支持 Markdown 正文中的 HTML/SVG mixed render。

不为了此需求迁移到 `markdown-it`、`remark/rehype` 或新渲染栈。

## 2. 当前链路

- `display-adapter.ts` 把消息正文整理成 markdown display block。
- `MessageBubble.vue` 渲染消息块。
- `MarkdownBlock.vue` 调用 `renderChatMarkdownResult(...)`。
- HTML/SVG/Mermaid fenced code block 仍走现有 preview 增强。

## 3. 策略

- 普通 Markdown 默认渲染。
- fenced `html` / `svg` / `mermaid` 继续走隔离 preview。
- raw HTML/SVG mixed render 只在明确开关允许时启用。
- sanitizer 分层处理 Markdown、raw HTML fragment、raw SVG fragment。
- 复杂 HTML 继续建议用 fenced code preview，不直接塞进正文 DOM。

## 4. 不迁库原因

当前项目已有：

- Markdown 主链路。
- Studio 媒体 token。
- copy button。
- Mermaid/HTML/SVG preview block。
- DOMPurify 安全规则。
- 全局/会话级开关。

迁库收益有限，改造面更大。

## 5. 验收

- 宽表格、长代码块、inline HTML、SVG 不撑宽 chat pane。
- raw HTML/SVG 关闭时继续转义显示。
- raw HTML/SVG 开启时不允许脚本、事件 handler、危险 URL 和样式污染。
- fenced preview 能继续全屏、复制和隔离渲染。
