## 1. Product State

The project has been reoriented from "OpenClaw management console" to "local-first AI Agent control workbench".

OpenClaw remains a supported runtime, but the durable Tracevane value is now:

- model gateway and provider protocol bridging
- CLI Agent workflow control
- workspace IDE for local edit/run/preview/verify loops
- IM/channel execution
- runtime recovery
- observability and evidence
- task-first human supervision

Product boundary reminder: OpenClaw is runtime substrate and platform support, not the center of the Tracevane product. New frontend work must be shaped around Tracevane's core jobs-to-be-done; OpenClaw-specific configuration, generic CRUD and host diagnostics stay under `/platforms/openclaw` or link out to official OpenClaw UI.

The broader product benchmark is now documented in `docs/product-benchmark-strategy.md`. Current product direction is to synthesize durable advantages from AI IDEs, app builders, observability/eval platforms, workflow automation and runtime managers without becoming a generic clone. New product design work should translate market lessons into one of the Tracevane pillars: Workspace IDE, Agent Runtime Control, Model Gateway, Channel Connectors, Evidence/Observability or System Guard/Recovery.

Tracevane is now the active product name and runtime id across package metadata, plugin id, UI copy, docs, generated client config defaults, release package names, installer script, gateway base path and `TRACEVANE_*` environment variables.

The old product naming compatibility layer has been intentionally removed from tracked code. Existing installs that still reference the previous plugin id, route prefix, provider id, custom media hint or environment variable names must be reinstalled or manually migrated to the Tracevane names.

The rename risk path has been hardened without restoring compatibility:

- `install-tracevane.sh` backs up retired extension directories before installing Tracevane.
- The installer prunes retired plugin ids, stale install records, stale load paths and stale plugin slot references from OpenClaw config, then writes only the Tracevane entry.
- Gateway service refresh now tries `openclaw gateway restart --safe` before falling back to plain restart, reducing the chance of interrupting active OpenClaw work.
- API/Web build and release packing clean generated output before compiling, so ignored `dist` residue cannot leak retired modules into a release tarball.

Dreaming / memory management and legacy plugin management are retired from Tracevane frontend/backend scope.
