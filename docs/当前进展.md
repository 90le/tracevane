## 1. Product State

The project has been reoriented from "OpenClaw management console" to "local-first AI Agent control workbench".

OpenClaw remains a supported runtime, but the durable Tracevane value is now:

- model gateway and provider protocol bridging
- CLI Agent workflow control
- workspace IDE for local edit/run/preview/verify loops
- IM/channel execution
- runtime recovery
- observability and evidence
- task-first human supervision

The broader product benchmark is now documented in `docs/product-benchmark-strategy.md`. Current product direction is to synthesize durable advantages from AI IDEs, app builders, observability/eval platforms, workflow automation and runtime managers without becoming a generic clone. New product design work should translate market lessons into one of the Tracevane pillars: Workspace IDE, Agent Runtime Control, Model Gateway, Channel Connectors, Evidence/Observability or System Guard/Recovery.

Tracevane is now the active product name and runtime id across package metadata, plugin id, UI copy, docs, generated client config defaults, release package names, installer script, gateway base path and `TRACEVANE_*` environment variables.

The old product naming compatibility layer has been intentionally removed from tracked code. Existing installs that still reference the previous plugin id, route prefix, provider id, custom media hint or environment variable names must be reinstalled or manually migrated to the Tracevane names.

The rename risk path has been hardened without restoring compatibility:

- `install-tracevane.sh` backs up retired extension directories before installing Tracevane.
- The installer prunes retired plugin ids, stale install records, stale load paths and stale plugin slot references from OpenClaw config, then writes only the Tracevane entry.
- Gateway service refresh now tries `openclaw gateway restart --safe` before falling back to plain restart, reducing the chance of interrupting active OpenClaw work.
- API/Web build and release packing clean generated output before compiling, so ignored `dist` residue cannot leak retired modules into a release tarball.

Dreaming / memory management and legacy plugin management are retired from Tracevane frontend/backend scope.

### Channel Connectors

- Codex, Claude Code and OpenCode are routed through structured persistent paths where available.
- Busy guard, file staging, permissions, progress, native compact and session evidence are covered by local and live smokes.
- TUI heartbeat parsing is only fallback evidence; official/structured events remain preferred.

### System Guard / Recovery

- OpenClaw 2026.6.8 config compatibility has been validated across web-write surfaces.
- Recovery can repair supported Gateway service and CLI package breakage on the current Linux/systemd-user environment.
- OpenClaw doctor warnings for plugin drift and obvious credential/config residue have been cleaned in the current environment.
- Remaining secret audit findings around agent model/auth plaintext keys are a separate migration risk, not an automatic repair path.

### Frontend Refocus

- Frontend redesign has entered implementation. `apps/web-vue` keeps the workspace path but now runs React + TypeScript + Vite 8 + Tailwind CSS v4; the old Vue source tree has been removed.
- Aurora is the active frontend baseline: `docs/prototypes/app.html`, `docs/prototypes/app/styles.css`, `docs/prototypes/app/pages.js`, `docs/prototypes/data/pages-data.js`, and 11 `docs/prototypes/pages/*.html` fragments are the visual/interaction source.
- The React shell maps Aurora into `route-manifest.ts`, `PrototypePage.tsx`, `page-mounts.ts`, and `AuroraShell.tsx`: left grouped navigation, light topbar, command palette, Sheet/Dialog/Toast, theme/palette switch, mobile nav, 11 hash routes and page-specific state flows.
- Current React implementation now has two page classes: 11 prototype-backed Aurora routes plus `/runtime-admin` as the first real React functional subdomain. Runtime Admin consumes existing system/config/agents/channels/skills/service/recovery APIs through TanStack Query and exposes OpenClaw support management behind secondary navigation.
- The frontend rebuild target is functional migration, not visual-only shelling. Prototype-backed pages remain temporary route/visual contracts until each page is replaced with real React components, existing API data, loading/empty/error states and focused system tests.
- Model Gateway and Channel Connectors pages have been simplified away from crowded two-column/all-fields layouts.
- Provider configuration now favors focused sections and drill-in editing.
- System sidebar label has been aligned to "System Status".
- Workspace IDE is now a core long-term domain, not just a maintenance terminal. The goal is a local edit/run/preview/AI/diff/approval workbench covering files, editor, terminal, Git, Markdown/HTML/web preview and evidence. See `docs/workspace-ide-goal.md`.

## 3. Retired Scope

Do not restore without a new PRD and user approval:

- Dreaming / memory management UI, BFF and shared types.
- Legacy plugin management UI, BFF and shared types.
- Old Gateway usage archive/pagination/CSV/cost/account/provider-key surface.
- Legacy platform action / third-party migration-first implementation tracks.

OpenClaw host/runtime plugin configuration and Recovery plugin-drift repair remain valid because they are runtime support, not legacy plugin management.
