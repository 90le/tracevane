# OpenClaw Studio 当前进展

> 状态：Active Snapshot
> 更新：2026-06-01

本文档是当前仓库状态摘要，面向后续开发接力。更细的产品原则见 `docs/产品需求.md`，架构边界见 `docs/系统架构.md`。

## 1. 总体状态

项目已从早期恢复阶段进入稳定迭代阶段：

- 管理域已经形成真实 API + Vue 工作台
- Chat 已从规划进入运行态
- Terminal、System、Channels、Agents、Skills、Cron、Config 等核心域具备真实读写能力
- Dashboard、Codex Stack、Files、Plugins、Dreaming 已有接入，但仍需要持续打磨信息架构和验收链路
- Room / Workflow 暂缓

## 2. 已稳定的主域

### Config

- 已支持 OpenClaw 配置读取、写入、patch、provider secret、channel secret 等能力
- 前端包含多个配置 tab 和高级配置 sheet
- 写操作需继续保持保守和可回滚

### Agents

- 已支持 Agent 列表、详情、创建、编辑、删除
- 已支持 docs、routing/bindings、sessions、advanced 配置页面
- 当前 workbench 仍处在布局与文案持续收口阶段

### Channels

- 已支持 provider、account、credential、access、pairing、routing/bindings 管理
- 支持 provider/account 深层路由和 drawer/sheet 交互
- 后续重点是继续统一“routing”文案，减少 binding 命名外泄

### Skills

- 已支持安装技能、市场浏览、预检、更新、删除、启停和配置/secret
- 已接入 marketplace source 管理

### Cron

- 已支持任务 CRUD、启停、手动运行、run history、delivery/failure 配置

### Terminal

- 已支持 PTY/WebSocket 维护终端
- 已支持 sessions、ledger、descriptor store、action catalog、recent output summary
- 也承担系统 handoff 和 CLI 安装/检查入口

### System

- 已支持 health、diagnostics、bootstrap repair、release/upgrade、runtime summary、device trust、event center、terminal handoff
- System event store / reader / writer / normalizer 已形成独立边界

## 3. Chat 当前状态

Chat 已经不是“尚未实现”的规划项。当前能力包括：

- `/api/chat/health`
- `/api/chat/bootstrap`
- organizer / folders / session assignment
- sessions list / create / patch / delete / reset
- history page / dates / search
- stream
- queue
- controls
- slash gateway
- send / abort
- upload
- resource resolve / media resolve

架构结论仍然是：

- Studio 后端做 typed adapter / BFF
- 浏览器不直连 OpenClaw Gateway
- Studio 自管 `studio_managed` 会话可写
- `observed_external` 默认只读
- 不引入第三方平行 DB 会话模型

前端当前已统一到一个目录：

- `features/chat/api.ts`：Studio Chat API client
- `features/chat/session-ref.ts`：route/session 引用 helper
- `features/chat/ChatShellPage.vue`：当前真实 Chat Shell
- `features/chat/*`：SessionList、ConversationPane、Composer、Inspector、Record Browser、Slash UI、富消息渲染等主要 UI

后续应继续保持单目录边界，避免重新出现版本化前端目录。

## 4. 仍需推进

### Dashboard

- 后端 summary 和前端消费链路已存在
- 仍需继续明确首屏核心指标、异常入口和跨域状态优先级

### Codex Stack

- 已有后端 service/routes 和前端 workspace
- 当前更像 guided operations workbench，后续重点是安装/修复/日志/模型路由的验收一致性

### Files

- 已有 browse、tree、read、search、download、upload、archive 等 API
- 前端已接入文件管理和编辑器
- 后续重点是权限边界、错误态和大文件体验

### Plugins

- 已有 plugin summary、toggle、install/update/uninstall、upload preflight 等 API
- 后续重点是 release/install metadata 与 UI 反馈一致性

### Dreaming

- 已纳入系统导航和 API
- 后续重点是定位清晰化：系统记忆/运行观测，而不是独立主产品线
