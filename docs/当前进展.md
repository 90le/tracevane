## 2. 当前主线

### Studio Gateway

- 独立 Gateway daemon、Provider Center、App Connections、active routing、模型目录、模型能力和上下文预算已形成主线。
- Gateway 目标是任意 provider 原生协议都能对外暴露 Anthropic Messages、OpenAI Responses / compact、OpenAI Chat Completions。
- `/tmp/cc-switch-src` 是协议转换成熟度参考。
- 真实 provider smoke 已覆盖 BigModel Chat、BigModel Anthropic、GMN Responses-native substitute；OpenAI Platform 官方端点保留为后续可选 vendor proof。
- 本地 BigModel/GLM provider 已加入 `glm-5.2`，按 1M context / 128K output 预算；Gateway 内置推断覆盖 `glm-5.2` 和 `glm-5.2[1m]`。
- Provider 已支持 endpoint profiles；本地 `glm` provider 配置 BigModel Coding Chat 与 Coding Anthropic 两个端点，同一模型会按客户端协议优选原生 endpoint，并按 endpoint health/circuit 回退；Provider Center 已支持 endpoint profile 编辑、detect 同步和端点级 smoke。
- Provider Center 已收口为任务分区式配置：基础连接、端点路由、密钥识别、模型目录、高级覆盖、可用范围；模型目录只暴露“模型名称”和“别名”两个身份字段，PC/平板/手机均已做响应式验证。
