## 1. 总体状态

OpenClaw Studio 已从早期恢复阶段进入真实工作台迭代：

- Config、Skills、Terminal、Channels、Cron、Agents、System 已具备真实读写和运行态能力。
- Chat 已进入运行态，继续保持 Studio typed adapter / BFF 边界。
- Studio Gateway 已成为正式模型中转域，旧 Codex Stack / CPA / Compact 用户链路已停止演进。
- Channel Connectors 已进入 Studio 原生 daemon 路线，当前聚焦 Feishu/Octo 私聊 + CLI Agent Bot。
- Dashboard、Files、Plugins、Dreaming 仍是持续打磨域。
- Room / Workflow 暂缓。

## 2. 当前主线

### Studio Gateway

- 独立 Gateway daemon、Provider Center、App Connections、active routing、模型目录、模型能力和上下文预算已形成主线。
- Gateway 目标是任意 provider 原生协议都能对外暴露 Anthropic Messages、OpenAI Responses / compact、OpenAI Chat Completions。
- `/tmp/cc-switch-src` 是协议转换成熟度参考。
- 真实 provider smoke 已覆盖 BigModel Chat、BigModel Anthropic、GMN Responses-native substitute；OpenAI Platform 官方端点保留为后续可选 vendor proof。

### Channel Connectors

- Studio 原生 Channel daemon 已可配置、启动、守护和查看运行态。
- Feishu/Octo 私聊文本、文件、图片、工具流、过程回复和核心命令已可用。
- active `studio-channel-skill` / platform action 方向已删除；平台 API 工具层不再扩展。
- 群聊、thread、多 bot 协作和管理类 Octo/Feishu action 只保留已实现 best-effort，不再作为当前主线。
- Claude Code / OpenCode persistent native compact 已有 driver 回归；Feishu 真实长连接 auto compact、Codex / Claude Code / OpenCode 显式 `/compact` 已验证进入 Agent-native compact；Codex / Claude / OpenCode 结构化工具输出、Codex reasoning summary 解析和 Octo `/thinking` 开关回归已补。
- Feishu 入站文件 live 已验证：文件消息 staging、history 附件摘要、Agent 读取文件和回复均通过。
- 图片自动切视觉模型默认关闭，需在平台 binding 或 IM `/vision` 显式开启；binding 可设置默认 fallback 模型，IM 会话可通过 `/vision` 命令和 Feishu 菜单临时开启、关闭、指定模型或回到 binding 默认。开启后失败会回退原模型的附件说明/本地路径模式。
- Codex 图片异常排查结果：当前 Feishu 私聊 override 实际为 `gpt-5.4-mini`；已修正 Codex `exec resume` 图片 option 顺序，并让 Codex app-server result 保留视觉 args，便于日志确认图片是否进入模型。
- Gateway 已修复 Responses -> Chat-compatible provider 的 `input_image` 映射；`gpt-5.4-mini` / `gpt-5.5` 受控图片 smoke 和 `codex exec --image` + `gpt-5.4-mini` 已通过。当前没有 `gpt-5.5-mini` 模型；`claude-opus-4-6` 走 `mlamp` Chat-compatible 图片请求仍返回 400，需按 provider smoke 结果标记能力。
- Codex / Claude Code / OpenCode 图片链路已用受控三色方块真实 runner smoke 验证通过；非视觉模型会收到附件说明和本地路径，不传 native 图片、不允许猜图。
- 视频附件不再由 Studio 预抽帧或转图片；Studio 只负责接收、staging 和把本地路径交给 Agent，由 Agent CLI/模型或工具按任务自行处理视频。
- Octo 媒体消息带 `content/caption` 时会保留用户任务文本，不再被 `[image]` / `[video]` 占位吞掉。
- 同 session FIFO 已补可恢复 pending store：已入队但尚未启动的 Agent 消息会落盘，daemon 重启后重放；daemon/API/UI 已展示 pending 数量、待恢复记录和最近 replay/drop/fail 事件；Octo restart 回归已通过，Feishu 路径待 live 复验。
- 真实 CLI thinking smoke 已完成：Claude Code 2.1.86 当前不输出 `thinking` item；OpenCode 1.17.0 在支持 reasoning 的模型上可输出 `reasoning` part；状态、菜单和前端已区分 parser/live 支持。
- 下一步聚焦 Agent runner 成熟度：工具流 live 差异、Feishu durable queue live 复验、真实 IM 文件/审批/compact 复验。

### Chat

- Chat 仍是 Studio 自管会话工作台，不直接暴露 Gateway raw frame。
- 后端 typed contract、会话列表、history、send、abort、reset、upload、stream 和 rich message 渲染已进入主线。
- 前端主实现集中在 `apps/web-vue/src/features/chat/**`。
- Chat 文档已压缩为合同和策略参考，避免继续追加性能流水账。
