Product boundary

把 Agent 运行的关键边界放到同一张工作台上。

OpenClaw 是当前支持的宿主和运行表面；Tracevane 的范围更大，覆盖本地工作区、模型网关、渠道连接器、CLI Agent、IDE 能力与系统守护。

01 / Workspace

Workspace / Files

文件操作、在线编辑、预览、终端交接和 Git/证据导向的项目工作流。

02 / Workbench

IDE Workbench

Explorer、Monaco 编辑器组、终端、Problems/Output、布局持久化与 Git/LSP 基础。

03 / Routing

Model Gateway

Provider 路由、协议适配、端点配置、账号媒体能力和活动路由可视化。

04 / Messaging

Channel Connectors

Feishu / Octo 私聊接入 Agent 工作流，保留权限、文件、进度和交付证据。

05 / Runtime

CLI Agents

Codex、Claude Code、OpenCode 运行准备度和 Agent Runs，而不是把终端管理伪装成 Agent。

06 / Guard

System Guard

OpenClaw 配置兼容、服务修复、守护健康、设备信任、运行摘要和审计事件。

Install

选择部署入口，复制给你的 Agent 执行。

为了避免 AI 或安全策略拦截，安装指令默认先下载脚本、可选审阅，再本地运行。正式发布站点会用 metadata 自动填充最新 Tracevane 版本与最低 OpenClaw 版本。

模式一：Standalone 非单口

适合本地访问，或可以直接开放 3760 端口的机器。路径清晰、排障直接。

模式二：Gateway 单口

适合外网只开放 OpenClaw 端口的环境，挂载到 /tracevane，同时保留本机 3760 回退入口。

独立端口 STANDALONE

请在我的 OpenClaw 环境里安装 Tracevane，使用非单口模式（standalone，独立端口 3760）。

请按下面步骤执行，不要使用在线脚本直通执行：

1. 先下载 https://tracevane.90le.cn/install-tracevane.sh 到本地当前目录
2. 可选执行：sed -n '1,160p' ./install-tracevane.sh
3. 执行：chmod +x ./install-tracevane.sh
4. 执行：./install-tracevane.sh --mode standalone

脚本会自动完成这些事情：
1. 检查并升级 OpenClaw 到最低要求 {{OPENCLAW_MIN_VERSION}}
2. 解析站点发布元数据并下载最新 Tracevane 安装包（当前 metadata 版本：{{TRACEVANE_VERSION}}）
3. 校验安装包 SHA-256，防止下载损坏或元数据不匹配
4. 自动修正发布包入口元数据，确保宿主能正确加载兼容插件入口
5. 安装依赖并重建 node-pty
6. 写入 plugins.entries.tracevane / plugins.load.paths / transport 配置
7. 执行 openclaw gateway install --force 与重启
8. 对 http://127.0.0.1:3760/api/system/health 做健康检查

执行完成后，请把以下结果告诉我：
- OpenClaw 版本
- 配置文件路径
- 安装目录
- 访问地址
- 健康检查结果

单口挂载 GATEWAY

请在我的 OpenClaw 环境里安装 Tracevane，使用单口模式（gateway，共用 OpenClaw 端口，挂载到 /tracevane，同时保留本机 3760 standalone 入口）。

请按下面步骤执行，不要使用在线脚本直通执行：

1. 先下载 https://tracevane.90le.cn/install-tracevane.sh 到本地当前目录
2. 可选执行：sed -n '1,160p' ./install-tracevane.sh
3. 执行：chmod +x ./install-tracevane.sh
4. 执行：./install-tracevane.sh --mode gateway

脚本会自动完成这些事情：
1. 检查并升级 OpenClaw 到最低要求 {{OPENCLAW_MIN_VERSION}}
2. 解析站点发布元数据并下载最新 Tracevane 安装包（当前 metadata 版本：{{TRACEVANE_VERSION}}）
3. 校验安装包 SHA-256，防止下载损坏或元数据不匹配
4. 自动修正发布包入口元数据，确保宿主能正确加载兼容插件入口
5. 安装依赖并重建 node-pty
6. 写入 plugins.entries.tracevane / plugins.load.paths / gateway transport 配置，并保留 3760 standalone 监听
7. 补齐 gateway.bind / gateway.auth.token / gateway.controlUi.allowedOrigins 等最低可用配置
8. 执行 openclaw gateway install --force 与重启
9. 对 http://127.0.0.1:<gateway_port>/tracevane/api/system/health 和 http://127.0.0.1:3760/api/system/health 做健康检查

执行完成后，请把以下结果告诉我：
- OpenClaw 版本
- 配置文件路径
- 安装目录
- Gateway 端口
- 3760 回退入口健康检查
- 访问地址
- 健康检查结果
