Tracevane本地 AI Agent 控制工作台

Tracevane 把本地项目、IDE Workbench、Model Gateway、CLI Agents、渠道连接器与恢复证据放到同一个可解释的操作平面。它不是聊天皮肤，也不伪装成全能仪表盘；它负责连接、观察、恢复、验证。

Product boundary

不做“AI 外壳”，只呈现 Tracevane 真正负责的边界。

参考顶级开发者工具页面的做法：把产品事实放在首位，用真实模块和真实安装路径建立信任，而不是堆叠无意义的发光卡片。

01 · Workspace / Files

本地工作区与文件表面。

文件操作、在线编辑、预览、终端交接和证据导向的项目工作流；不会把文件管理器伪装成聊天页。

02 · IDE Workbench

真实 IDE 工作台边界。

Explorer、Monaco 编辑器组、Terminal、Problems / Output、布局持久化与 Git/LSP 基础。

03 · Model Gateway

模型路由与协议适配。

Provider 路由、端点配置、账号媒体能力和活动路由可观察，而不是黑盒转发。

04 · Channel Connectors

IM 渠道到 Agent 工作流。

Feishu / Octo 私聊接入，保留权限、文件、进度和交付证据。

05 · CLI Agents / System Guard

运行准备度、系统守护与恢复证据。

Codex、Claude Code、OpenCode 的 readiness 和 Agent Runs；OpenClaw 配置兼容、服务修复、健康检查和审计事件。

Operating model

四个动作，比一堆营销词更重要。

连接运行面，观察真实状态，恢复故障入口，验证每一次安装和交付。页面动效只服务于信息分层：短、准、可关闭、尊重 reduced motion。

Install

选择部署入口，复制给你的 Agent 执行。

为了避免 AI 或安全策略拦截，安装指令默认先下载脚本、可选审阅，再本地运行。正式发布站点会用 metadata 自动填充最新 Tracevane 版本与最低 OpenClaw 版本。

Standalone 非单口适合本地访问，或可以直接开放 3760 端口的机器。
Gateway 单口挂载挂载到 /tracevane，同时保留本机 3760 回退入口。

独立端口 STANDALONE

请在我的 OpenClaw 环境里安装 Tracevane，使用非单口模式（standalone，独立端口 3760）。

请按下面步骤执行，不要使用在线脚本直通执行：

1. 先下载 https://github.com/90le/tracevane/releases/latest/download/install-tracevane.sh 到本地当前目录
2. 可选执行：sed -n '1,160p' ./install-tracevane.sh
3. 执行：chmod +x ./install-tracevane.sh
4. 执行：./install-tracevane.sh --check-release
5. 确认来源、版本和 SHA-256 后执行：./install-tracevane.sh --mode standalone --json

脚本会自动完成这些事情：
1. 检查并升级 OpenClaw 到最低要求 {{OPENCLAW_MIN_VERSION}}
2. 解析站点发布元数据并下载最新 Tracevane 安装包（当前 metadata 版本：{{TRACEVANE_VERSION}}）
3. 校验安装包 SHA-256，防止下载损坏或元数据不匹配
4. 自动修正发布包入口元数据，确保宿主能正确加载兼容插件入口
5. 安装依赖并重建 node-pty
6. 写入 plugins.entries.tracevane / plugins.load.paths / transport 配置
7. 执行 openclaw gateway install --force 与重启
8. 对 http://127.0.0.1:3760/api/system/health 做健康检查

执行完成后，请原样返回 JSON，并总结 version、installDir、configPath、accessUrls、healthChecks、warnings、degradedFeatures；不要使用 curl | bash，也不要绕过 checksum、TLS、OpenClaw 配置校验或健康检查。
执行完成后，请把以下结果告诉我：
- OpenClaw 版本
- 配置文件路径
- 安装目录
- 访问地址
- 健康检查结果

单口挂载 GATEWAY

请在我的 OpenClaw 环境里安装 Tracevane，使用单口模式（gateway，共用 OpenClaw 端口，挂载到 /tracevane，同时保留本机 3760 standalone 入口）。

请按下面步骤执行，不要使用在线脚本直通执行：

1. 先下载 https://github.com/90le/tracevane/releases/latest/download/install-tracevane.sh 到本地当前目录
2. 可选执行：sed -n '1,160p' ./install-tracevane.sh
3. 执行：chmod +x ./install-tracevane.sh
4. 执行：./install-tracevane.sh --check-release
5. 确认来源、版本和 SHA-256 后执行：./install-tracevane.sh --mode gateway --json

脚本会自动完成这些事情：
1. 检查并升级 OpenClaw 到最低要求 {{OPENCLAW_MIN_VERSION}}
2. 解析站点发布元数据并下载最新 Tracevane 安装包（当前 metadata 版本：{{TRACEVANE_VERSION}}）
3. 校验安装包 SHA-256，防止下载损坏或元数据不匹配
4. 自动修正发布包入口元数据，确保宿主能正确加载兼容插件入口
5. 安装依赖并重建 node-pty
6. 写入 plugins.entries.tracevane / plugins.load.paths / gateway transport 配置，并保留 3760 standalone 监听
7. 补齐 gateway.bind / gateway.auth.token / gateway.controlUi.allowedOrigins 等最低可用配置
8. 执行 openclaw gateway install --force 与重启
9. 对 http://127.0.0.1:<gateway_port>/tracevane/api/system/health 和 http://127.0.0.1:3760/api/system/health 做健康检查

执行完成后，请原样返回 JSON，并总结 version、installDir、configPath、accessUrls、healthChecks、warnings、degradedFeatures；确认 3760 fallback health，禁止 curl | bash 或绕过 checksum、TLS、配置校验和健康检查。
执行完成后，请把以下结果告诉我：
- OpenClaw 版本
- 配置文件路径
- 安装目录
- Gateway 端口
- 3760 回退入口健康检查
- 访问地址
